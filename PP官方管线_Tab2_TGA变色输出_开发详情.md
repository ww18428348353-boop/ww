# PP官方管线 Tab 2 — TGA变色输出 开发详情文档

> 模块位置: `modules/pp_pipeline_page.py` → `PPExportPage` (继承 `PPPipelinePage`)
> 容器: `PPPipelineTabsPage` (Tab 1: PP调色 / **Tab 2: TGA变色输出**)
> 版本: PP_Pipeline v1.0.0

---

## 一句话概括

**加载 TGA/PNG 序列帧 → 14 字段 PP 官方管线实时预览（双画布对比: 本体 vs 变色后）→ 多方案管理 → 批量导出变色 PNG 序列。**

核心用途: 给召唤兽/角色 TGA 序列帧批量应用 PP 调色参数后输出为 PNG, 用于 TCP 替换/合图流水线的素材准备。

## PP 调色管线 — 算法核心

### 变换流水线 `_pipeline_rgb` (modules/pp_core.py)

```
输入: (H, W, 3) float32 RGB, 值域 [0, 255]
    │
    ▼
[步骤1] 3×3 通道混合矩阵 (Q256 定点)
    │   M = [[rr,rg,rb],[gr,gg,gb],[br,bg,bb]] / 256.0
    │   rgb_new = rgb @ M.T
    │   clip(0, 255)
    ▼
[步骤2] RGB → HSL → 修改 H/S/L → RGB
    │   H = (H + h_slider/500 * 360) mod 360
    │   S = clip(S * s_slider/500, 0, 1)
    │   L = clip(L * l_slider/500, 0, 1)
    │   (仅 h≠0 或 s≠500 或 l≠500 时启用)
    ▼
[步骤3] 线亮度 (Q100)
    │   rgb *= lb / 100.0
    ▼
[步骤4] 对比度 (Q128)
    │   rgb = (rgb - 128) * c/128 + 128
    ▼
输出: clip(0, 255) → uint8
```

---

## ⭐ 重点: 为什么影子部分不受变色影响

### 原因分析

PP 官方管线的 14 字段变色**只处理 RGB 像素值**, 对 Alpha 通道完全不做任何修改。关键点在于:

#### 1. Alpha 通道原样保留

在 `pp_core.apply_pp_adjust()` 中:

```python
rgb   = img[..., :3].astype(np.float32)   # 只取 RGB
alpha = img[..., 3:4].astype(np.float32)   # Alpha 独立保存

rgb = _pipeline_rgb(rgb, adj)              # 只对 RGB 做变换

out = np.concatenate([rgb, alpha], axis=-1) # Alpha 原封不动拼回
```

**Alpha 从头到尾没有经过任何变换管线。**

#### 2. 影子的像素特征

在大话二代 TGA 序列帧中, "影子" 通常有以下特征:

- **RGB 值极低 (接近黑色)**: 影子区域的 R≈0, G≈0, B≈0 (或非常接近 0)
- **Alpha 为半透明**: 影子不是纯黑实心, 而是用半透明黑色叠加到背景上

#### 3. PP 管线对黑色/暗色像素的变换效果趋近于零

分析每个步骤对黑色像素 (R=0, G=0, B=0) 的影响:

| 步骤 | 公式 | 对 (0,0,0) 的结果 | 结论 |
|------|------|-------------------|------|
| 步骤1: 通道混合 | `rgb @ M.T` | (0,0,0) × 任何矩阵 = (0,0,0) | **无影响** |
| 步骤2: HSL 修改 | RGB→HSL | R=G=B=0 → H=0, S=0, **L=0** | **无影响** |
| | H 偏移 | 任何偏移对 S=0 的灰色像素无效 | 色相偏移对无饱和度像素不起作用 |
| | S 缩放 | S=0 × 任何系数 = 0 | 饱和度调整对灰色/黑色无效 |
| | L 缩放 | L=0 × 任何系数 = 0 | 亮度 0 乘以任何值还是 0 |
| 步骤3: 线亮度 | `rgb × lb/100` | 0 × 任何值 = 0 | **无影响** |
| 步骤4: 对比度 | `(rgb-128)×c/128+128` | (0-128)×c/128+128 | **⚠ 唯一可能有影响的步骤** |

#### 4. 对比度步骤的特殊情况

对比度步骤 `(0-128) × c/128 + 128`:
- 当 c=128 (中性): (-128)×1+128 = 0 → 无变化
- 当 c≠128: 黑色像素会被轻微拉向 128 (c<128) 或推向 0 以下被 clip (c>128)

但实际上:
- **大多数变色方案对比度保持在 128 附近** (100~180 范围)
- 即使对比度不是 128, 对极暗像素的影响也非常小
- 结果被 `clip(0, 255)` 截断, RGB=0 在大部分情况下还是 0 或极小值

#### 5. 综合结论: 影子不受变色影响的根本原因

```
┌─────────────────────────────────────────────────────────────┐
│ 影子 = RGB 极暗 (接近 0,0,0) + Alpha 半透明                 │
│                                                             │
│ ① 矩阵混合: 黑色乘任何矩阵还是黑色 (0×anything=0)          │
│ ② HSL: 黑色的 L=0, S=0 → 色相/饱和度/亮度调整全部无效      │
│ ③ 线亮度: 0 乘以任何系数还是 0                              │
│ ④ 对比度: 对极暗像素影响极小, 且被 clip 截断               │
│                                                             │
│ 结果: 影子像素经过完整 4 步管线后仍然≈(0,0,0,原Alpha)       │
│ → 视觉上影子颜色/透明度完全不变                              │
└─────────────────────────────────────────────────────────────┘
```

**简单说: PP 官方管线是乘性变换为主的管线, 黑色是乘法的"零元" — 任何乘性操作对黑色都无效。影子因为 RGB≈0, 天然免疫了通道混合 + HSL + 线亮度三个步骤, 只有对比度有微弱影响但也被 clip 抹掉了。这是管线数学特性决定的, 不是特意做的"跳过影子"逻辑。**

## 导出线程 `_TgaExportWorker`

### 两种模式

| 模式 | 触发条件 | 输出结构 |
|------|---------|---------|
| 平铺模式 | 单目录 / 只有"序列"一个动作 | `<out>/1000.png, 1001.png, ...` |
| 保持结构模式 | 有多个子目录动作 | `<out>/<action>/1000.png, 1001.png, ...` |

### 处理流程 (每张图)

```
tga_path → load_tga() → QImage
    → convertToFormat(RGBA8888)
    → qimage_to_rgba() → numpy (H,W,4)
    → pp_core.apply_pp_adjust(arr, adj)  ← 14 字段管线
    → rgba_to_qimage() → QImage
    → .save(out_fp, "PNG")
```

### 线程信号

| Signal | 参数 | 用途 |
|--------|------|------|
| `progress` | (done, total, cur_name) | 更新进度条 |
| `message` | (text, level) | 写入日志 (info/warn/error) |
| `finished_ok` | (done, total, out_dir) | 正常完成 |
| `failed` | (text) | 异常中止 |

## 与 Tab 1 的差异

| 特性 | Tab 1 (PP调色) | Tab 2 (TGA变色输出) |
|------|----------------|-------------------|
| 画布 | 单画布 (变色后) | **双画布** (本体 + 变色后) |
| 素材类型 | TCP 或 TGA | **TGA/PNG 序列** (不支持 TCP) |
| 多动作 | 单一占位 | **自动扫描子目录** 作为动作 |
| 批量导出 | 无 | **✅ 后台线程 PNG 批量输出** |
| PNG保存PP | 无 | **✅ 按源目录名自动命名** |
| 文件名支持 | .tga | **.tga + .png** |
