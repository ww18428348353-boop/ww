# 已知问题 / 后期优化清单

> 用于记录"功能层基本可用,等全部模块完成后再统一打磨"的事项.
> 优先级: **后期优化** (M9~M10 阶段统一处理,不阻塞 M4~M8 功能开发).

---

## 1. 颜色精度对齐 AE (HSV/HSB 微调)  〔最后优化的点〕

**现状**: M4 effect_chain.hlsl 的色相/饱和度已从 HSL 改为 HSV/HSB,主体方向与 AE 一致,但仍有视觉差异:

- 白衣不够白 / 整体偏暗
- 高饱和区与 AE 略有色偏
- 色相旋转后的色彩对比度比 AE 弱

**可能原因**:

1. AE 内部全程 32bpc Linear 浮点工作流,我们用 sRGB 8bpc 数值
2. 半透 alpha 边缘合成方式 (linear over vs sRGB over) 累积误差
3. `applyHsl` 加性饱和度公式 与 AE 内部曲线不完全一致 (AE 在 saturation 负值时可能用更强的衰减)
4. 白点处理 (V = 1 时) 的边界细节
5. sRGB 工作流: 当前是 `RTV = SRGB`,GPU 自动 linear→sRGB encode,但 Sample 源帧时未做 sRGB→linear (源帧 SRV 仍 UNORM)

**计划处理时机**: M9 (性能调优) 阶段一并 sRGB 工作流统一升级:

- [ ] 源帧 SRV 改为 `R8G8B8A8_UNORM_SRGB` → GPU 自动 sRGB→linear
- [ ] 所有效果链全程 linear 空间计算
- [ ] 输出仍走 SRGB RTV (linear→sRGB 自动)
- [ ] 与 AE 32bpc Linear 工作流完全等价

**测试用例**: 截图三 AE vs 我们 (色相+170 / 饱和度-40 / 亮度0) 像素级误差 < 5/255

## 2. 曲线编辑器 UI 占位

**现状**: M4 批 1 中 `EffectPanel` 的 "3. 曲线" 段只放一个 placeholder 标签 (M6 给完整曲线编辑器)。但 HLSL `applyCurves` + `CurveSolver` 已实现,后台 LUT 256x4 也已上传,只是 UI 不能改控制点。

**计划处理时机**: M6 (复杂控件) 阶段补完整曲线编辑器:

- [ ] 256×256 自绘画布
- [ ] 左键添加 / 拖动 / 右键删除控制点
- [ ] RGB 主曲线 + R/G/B 三通道分别编辑切换
- [ ] 实时刷新 LUT 纹理

## 3. 通道混合器 / 色彩平衡 简化版

**现状**: 
- 通道混合器: 9 滑块全开,可用,但缺"输出通道单选" + "单色按行系数"等 AE 完整 UX
- 色彩平衡: 9 滑块 (3 区 × 3 通道) 已完整,只是 UI 没用 AE 风格的"青↔红 / 品↔绿 / 黄↔蓝"色块滑块

**计划处理时机**: M6 完整复杂控件实现

## 4. windeployqt 中文路径中转

**现状**: build.bat 的 windeployqt 步骤先把 EXE 拷到 `%TEMP%\HighPro_LUT_deploy` (英文路径) 再调 windeployqt,然后拷回。这是规避 Qt windeployqt 工具链不支持中文路径的 bug。

**计划处理时机**: M10 切换 Qt 静态版后,这个中转步骤可去掉 (静态链接无需运行时 dll)。

---

## 5. FrameLoader LRU 策略简陋

**现状**: M2 简易缓存 — 当 `m_cache.size() >= maxEntries` 时清掉一半。`QHash::keys()` 顺序无定义,可能淘汰活跃 LUT。

**计划处理时机**: M9 升级为真正 LRU + 异步加载 (3 IO 线程 + PBO 上传)

## 7. 影子保护阈值默认值

**现状**: `EffectStack::shadowProtectThreshold = 8` 写死,UI 没暴露。

**计划处理时机**: M7 (随机/重置/肤色层) 阶段在 EffectPanel 顶部加"高级"折叠组,放阈值滑块。

## 8. 老 LUT 路径 (Ctrl+1..6) 与 7 效果链冲突

**现状**: `FrameRenderer::render()` 中 useEffect 优先级高于 useLut。当用户既启用 7 效果又加载了 add_lut PNG 时,LUT 被忽略。

**预期行为**: 这个优先级是**对的** (7 效果是主路径,LUT 仅参考用)。但 UI 上需要在加载 LUT 后显示提示"7 效果链优先,清除效果后 LUT 才生效"。

**计划处理时机**: M8 导出菜单一并完善状态栏提示

---

## 9. addon 默认隐藏除第一个外的子层

**现状**: `ProjectController::loadSource` 默认把第一个 addon 之外的 addon 设隐藏。

**问题**: 用户切换不同方案时希望能保持自己的 addon 选择,不应每次加载都重置。

**计划处理时机**: M8 项目持久化时一并处理 (从 .json 加载用户上次的 addon 选择)。
