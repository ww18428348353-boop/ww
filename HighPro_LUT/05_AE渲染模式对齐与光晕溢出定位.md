# HighPro_LUT — AE 渲染模式对齐与 TGA 光晕溢出定位

> 生成日期: 2026-06-02  
> 项目根: `X:\Lut变色\HighPro_LUT`  
> 问题对象: 同一张带 Alpha 通道的 `0000.tga`，在 AE / 游戏 / 开发 EXE 中显示结果不一致，开发 EXE 出现光晕外扩、溢出、偏亮。

---

## 1. 问题现象

同一张 TGA 带 Alpha 通道素材，在三种环境里显示：

```text
AE  →  游戏  →  开发 EXE
```

AE 和游戏表现基本一致。  
开发 EXE 中，角色周围高亮光效区域明显更亮，光晕范围更大，视觉上像光晕溢出。

该问题不是简单的颜色偏差，而是 **半透明高亮区域的合成结果不一致**。

---

## 2. 当前 EXE 渲染管线摘要

当前 EXE 管线来自 `03_TGA加载与渲染管线.md`：

```text
TGA 文件，Straight RGBA，sRGB 字节
  ↓
stbi_load_from_memory(req=4)
  ↓
R8G8B8A8_UNORM 纹理
  ↓
Shader Sample，得到 sRGB 数值
  ↓
Shader 手动 srgbToLinear()
  ↓
PS 输出 linear RGB + 原 alpha
  ↓
BlendState = SRC_ALPHA / INV_SRC_ALPHA
  ↓
B8G8R8A8_UNORM_SRGB RTV
  ↓
GPU 写 RTV 时 linear → sRGB encode
  ↓
Present
```

核心状态：

| 模块 | 当前配置 |
|---|---|
| TGA 解码 | `stbi_load_from_memory(..., req=4)` |
| 源纹理格式 | `DXGI_FORMAT_R8G8B8A8_UNORM` |
| 源纹理 Alpha | Straight Alpha，不预乘 |
| Shader | 手动执行 `srgbToLinear()` |
| BlendState | `SRC_ALPHA / INV_SRC_ALPHA` |
| RenderTargetView | `DXGI_FORMAT_B8G8R8A8_UNORM_SRGB` |
| 合成空间 | Linear space |

---

## 3. 初步结论

开发 EXE 的光晕溢出，主要原因是：

```text
EXE 当前在 Linear 色彩空间做 Alpha Blend，
而 AE / 游戏表现更接近 sRGB / Gamma 空间 Alpha Blend。
```

也就是说，问题不是 TGA 文件坏了，也不是解码错了，而是 **颜色空间和 Alpha 合成模型没有和 AE / 游戏保持一致**。

---

## 4. 为什么 Linear Blend 会让光晕更亮

光晕类素材常见像素特征：

```text
RGB 很亮
Alpha 很低
```

例如：

```text
src.rgb = 高亮黄色 / 高亮青色
src.a   = 0.05 ~ 0.20
```

### 4.1 sRGB / Gamma 空间合成

```text
out_srgb = src_srgb * alpha + dst_srgb * (1 - alpha)
```

该模式下，低 Alpha 的高亮像素会被压暗得比较明显。  
视觉结果通常更接近 AE 默认 8bpc 工程和很多 2D 游戏引擎 Sprite 渲染。

### 4.2 Linear 空间合成

```text
src_linear = srgbToLinear(src_srgb)
dst_linear = srgbToLinear(dst_srgb)
out_linear = src_linear * alpha + dst_linear * (1 - alpha)
out_srgb   = linearToSrgb(out_linear)
```

该模式更物理正确，但对光效边缘会保留更多能量。  
结果是：

```text
低 Alpha 高亮像素更容易显现
光晕边缘更亮
视觉范围更大
看起来像外扩 / 溢出
```

这和当前截图中开发 EXE 的表现一致。

## 5. 代码级定位

### 5.1 RTV 使用了 `_SRGB`

文件：

```text
src/ui/D3DWidget.cpp
```

当前代码：

```cpp
rtvDesc.Format = DXGI_FORMAT_B8G8R8A8_UNORM_SRGB;
```

该配置表示 RenderTargetView 是 sRGB 视图。  
Shader 输出会被当作 linear RGB，写入时 GPU 自动进行 linear → sRGB encode。

---

### 5.2 Shader 手动做 `srgbToLinear()`

文件：

```text
assets/shaders/fullscreen_quad.hlsl
assets/shaders/recolor.hlsl
assets/shaders/effect_chain.hlsl
```

典型代码：

```hlsl
float4 c = gTex.Sample(gLinearClamp, i.uv);
if (c.a < 0.001) {
    return float4(0, 0, 0, 0);
}
c.rgb = srgbToLinear(saturate(c.rgb));
return c * tint;
```

以及：

```hlsl
mapped = srgbToLinear(mapped);
return float4(mapped, src.a) * tint;
```

这说明 EXE 当前明确把 TGA 的 sRGB 字节值转到 linear 空间输出。

---

### 5.3 BlendState 本身没有明显错误

文件：

```text
src/render/FrameRenderer.cpp
```

当前代码：

```cpp
bd.RenderTarget[0].SrcBlend  = D3D11_BLEND_SRC_ALPHA;
bd.RenderTarget[0].DestBlend = D3D11_BLEND_INV_SRC_ALPHA;
bd.RenderTarget[0].BlendOp   = D3D11_BLEND_OP_ADD;
```

该公式是标准 Straight Alpha over：

```text
out.rgb = src.rgb * src.a + dst.rgb * (1 - src.a)
out.a   = src.a         + dst.a   * (1 - src.a)
```

这部分和 Straight Alpha 素材匹配。  
当前问题不是 BlendState 公式错，而是 **这个公式在 Linear 空间执行，和 AE / 游戏默认表现不同**。

---

## 6. 是否和开发环境有关

结论：基本无关。

| 项 | 是否主因 | 说明 |
|---|---:|---|
| Visual Studio / MSVC | 否 | 编译器不会改变 D3D blend 颜色空间语义 |
| Qt 版本 | 否 | Qt 只负责窗口承载，不决定 shader / RTV 合成模型 |
| Windows 版本 | 否 | D3D11 行为稳定，按当前状态执行 |
| NVIDIA / Intel GPU | 通常否 | 可能有极小浮点差异，但不会造成明显光晕外扩 |
| 显示器 ICC / HDR | 可能影响观感 | 但不是本问题主要原因 |
| RTV 格式 `_SRGB` | 是 | 决定写入和 blend 的颜色空间 |
| Shader `srgbToLinear()` | 是 | 明确把 sRGB 素材转 linear |
| Blend 颜色空间 | 是 | 直接影响半透明高亮像素结果 |

---

## 7. 和 AE 保持一致的建议模式

如果目标是匹配截图中的 AE / 游戏效果，建议 EXE 切换为 **sRGB / Gamma 空间合成模式**。

目标管线：

```text
TGA sRGB 字节
  ↓
R8G8B8A8_UNORM 纹理
  ↓
Shader 直接使用 sRGB 数值，不做 srgbToLinear
  ↓
PS 输出 sRGB RGB + 原 alpha
  ↓
BlendState = SRC_ALPHA / INV_SRC_ALPHA
  ↓
B8G8R8A8_UNORM RTV
  ↓
Present
```

目标合成公式：

```text
out_srgb = src_srgb * alpha + dst_srgb * (1 - alpha)
```

## 8. 推荐修改方案

### 8.1 修改 RTV 格式

文件：

```text
src/ui/D3DWidget.cpp
```

当前：

```cpp
rtvDesc.Format = DXGI_FORMAT_B8G8R8A8_UNORM_SRGB;
```

建议改为：

```cpp
rtvDesc.Format = DXGI_FORMAT_B8G8R8A8_UNORM;
```

---

### 8.2 移除透传 Shader 中的 `srgbToLinear()`

文件：

```text
assets/shaders/fullscreen_quad.hlsl
```

当前：

```hlsl
float4 c = gTex.Sample(gLinearClamp, i.uv);
if (c.a < 0.001) {
    return float4(0, 0, 0, 0);
}
c.rgb = srgbToLinear(saturate(c.rgb));
return c * tint;
```

建议改为：

```hlsl
float4 c = gTex.Sample(gLinearClamp, i.uv);
if (c.a < 0.001) {
    return float4(0, 0, 0, 0);
}
return saturate(c) * tint;
```

---

### 8.3 移除 LUT Shader 中的 `srgbToLinear()`

文件：

```text
assets/shaders/recolor.hlsl
```

当前：

```hlsl
mapped = srgbToLinear(mapped);
return float4(mapped, src.a) * tint;
```

建议改为：

```hlsl
return float4(mapped, src.a) * tint;
```

---

### 8.4 移除 Effect Shader 中的 `srgbToLinear()`

文件：

```text
assets/shaders/effect_chain.hlsl
```

当前：

```hlsl
c = srgbToLinear(c);
return float4(c, src.a) * tint;
```

建议改为：

```hlsl

```

---

### 8.5 清屏颜色不要转 Linear

文件：

```text
src/render/FrameRenderer.cpp
```

当前：

```cpp
const float bg[4] = {
    (float)sRgbToLinear(bgColor.redF()),
    (float)sRgbToLinear(bgColor.greenF()),
    (float)sRgbToLinear(bgColor.blueF()),
    1.0f,
};
```

建议改为：

```cpp
const float bg[4] = {
    (float)bgColor.redF(),
    (float)bgColor.greenF(),
    (float)bgColor.blueF(),
    1.0f,
};
```

---

### 8.6 BlendState 保持不变

继续使用：

```cpp
SrcBlend  = D3D11_BLEND_SRC_ALPHA;
DestBlend = D3D11_BLEND_INV_SRC_ALPHA;
```

不要改成：

```cpp
SrcBlend  = D3D11_BLEND_ONE;
DestBlend = D3D11_BLEND_INV_SRC_ALPHA;
```

除非整个纹理管线同步改成 Premultiplied Alpha。  
当前素材从统计看是 Straight Alpha，盲目切 Premultiplied Blend 会导致更亮、更炸。

---

## 9. 第二可疑点：缩放采样导致 Straight Alpha 边缘污染

当前采样逻辑：

```cpp
const bool oneToOne = (wpx == tw && hpx == th);
ID3D11SamplerState* frameSampler =
    oneToOne ? m_samplerPoint.Get() : m_samplerLinear.Get();
```

也就是：

```text
1:1 显示 → POINT
缩放显示 → LINEAR
```

Straight Alpha 纹理在 LINEAR 过滤时可能出现边缘污染：

```text
RGB 和 Alpha 独立插值
透明边缘 RGB 仍然很亮
缩放时亮色被插值出来
边缘出现额外光晕
```

因此需要区分两个问题：

| 测试 | 判断 |
|---|---|
| 100% zoom + POINT 后光晕明显减少 | 存在 Straight Alpha 线性过滤问题 |
| 改 Gamma pipeline 后光晕接近 AE / 游戏 | 主因是 Linear blend 不一致 |
| 两者都影响 | 需要同时修颜色空间和采样策略 |

---

## 10. 建议调试步骤

### Step 1：固定 100% 缩放

确保 `wpx == tex_w && hpx == tex_h`，强制走 POINT。  
观察光晕是否仍然比 AE / 游戏大。

### Step 2：临时改 RTV 为 UNORM

只改：

```cpp
DXGI_FORMAT_B8G8R8A8_UNORM_SRGB
```

为：

```cpp
DXGI_FORMAT_B8G8R8A8_UNORM
```

同时临时注释 Shader 里的 `srgbToLinear()`。  
这是验证主因最快路径。

### Step 3：同背景色取样比较

用同样背景色，在 AE 和 EXE 中取同一像素附近的 RGB。  
如果差异集中在半透明光晕区域，说明 Alpha 合成空间就是主因。

### Step 4：检查 AE 素材解释

AE 中确认 TGA：

```text
Interpret Footage > Alpha > Straight - Unmatted
```

如果 AE 解释为 Premultiplied，而 EXE 当 Straight 用，也会有边缘差异。  
但从当前素材统计看，更符合 Straight Alpha。

### Step 5：确认 AE 工程颜色设置

需要确认 AE 工程是否启用：

```text
Linearize Working Space
```

如果 AE 默认 8bpc / 非 Linear，EXE 应使用 Gamma-space blend。  
如果 AE 开了 Linear 工作流，EXE 当前 linear pipeline 才更接近。

---

## 11. 最终建议

如果目标是和截图中的 AE / 游戏一致，建议采用：

```text
R8G8B8A8_UNORM 源纹理
B8G8R8A8_UNORM RTV
Shader 不做 srgbToLinear
BlendState 保持 SRC_ALPHA / INV_SRC_ALPHA
ClearColor 直接使用 sRGB 值
1:1 使用 POINT
缩放时根据需求评估 LINEAR 或 alpha-aware 过滤
```

核心目标：

```text
让 EXE 的合成公式变成：
out_srgb = src_srgb * alpha + dst_srgb * (1 - alpha)
```

这样更接近 AE 默认预览和多数 2D 游戏 Sprite 管线。

---

## 12. 结论一句话

开发 EXE 光晕溢出不是 TGA 导入错误，也不是开发环境问题，而是当前 EXE 使用了 `srgbToLinear()` + `_SRGB RTV` 的 Linear Alpha Blend 管线；要匹配 AE / 游戏，应改为 sRGB / Gamma 空间 Straight Alpha 合成。
