# 随机变色深度总结与优化建议

> 项目: `HighPro_LUT`  
> 参考文档: `颜色随机与效果空间详解.md`  
> 目标: 优化当前 EXE 随机变色质量，使 27 套随机方案从“参数随机”升级为“设计感配色”。

---

## 一、核心结论

当前 EXE 随机变色不好看，主因不是效果链能力弱，而是随机策略太“参数级随机”。

AE 示例更像：

```text
先定整体配色主题
→ 再按角色部件分配颜色
→ 最后微调明暗、材质、发光
```

当前 EXE 更像：

```text
每个层独立随机一套 HSL / 亮度 / 颜色平衡 / 滤镜参数
```

因此会出现：

- 颜色之间缺少主次关系。
- 每个层都在抢视觉焦点。
- 高饱和霓虹色过多。
- 发光、衣服、坐骑、装饰互相串色。
- 暗部、阴影、金属、皮肤、布料缺少材质保护。
- 27 个方案看起来像“随机染色集合”，不像“设计过的皮肤方案”。

优化核心：

```text
从“随机效果参数”升级为“随机设计方案”
```

---

## 二、当前随机系统总结

根据 `颜色随机与效果空间详解.md`，当前系统底层能力完整，效果链足够支撑高质量换色。

### 2.1 效果链结构

当前有 7 个 AE 对齐效果：

1. `色相/饱和度`
2. `亮度/对比度`
3. `曲线`
4. `通道混合器`
5. `颜色平衡`
6. `照片滤镜`
7. `自然饱和度`

外加一个 `影子保护`，防止纯黑、极暗区域被染脏。

这套效果链能力够强。问题主要在随机参数生成策略。

### 2.2 当前随机逻辑

核心函数：

```cpp
randomizeStack(EffectStack& s, quint32 seed)
```

随机过程先抽一个风格档位：

| 档位 | 概率 | 特征 |
|---|---:|---|
| Realistic | 70% | 低饱和，接近写实 |
| Dark | 20% | 压暗，脱色 |
| Natural | 8% | 稍微自然增强 |
| Vivid | 2% | 高饱和创意色 |

然后逐个效果随机参数。

当前最大问题：

```text
HSL 色相：-180 到 180 全范围随机
随机所有层：默认每层独立 seed
```

这会导致每个层被扭到完全不同色相。角色有坐骑、衣服、发光、武器、装饰多个层时，整体马上碎掉。

## 三、AE 示例为什么好看

前七张 AE 示例里的变色效果不是纯随机，而是“受控变色”。

### 3.1 颜色数量少

通常只出现：

- 主色 1 个
- 辅色 1 个
- 高光色 1 个
- 暗部色 1 个
- 金属 / 白 / 黑保留色

不是所有层都随机不同颜色。

### 3.2 明度结构保留

AE 变色多数保持原始明暗关系：

- 亮的地方还是亮。
- 暗的地方还是暗。
- 高光不乱染。
- 阴影不变脏。
- 轮廓仍清楚。

当前 EXE 部分方案发灰、发脏、发花，就是因为色相随机和饱和度随机叠加后破坏了明度关系。

### 3.3 发光色统一

AE 示例里的发光一般很统一：

- 蓝色方案：发光蓝 / 青。
- 粉色方案：发光粉 / 紫。
- 金色方案：发光黄 / 暖白。
- 黑白方案：发光白 / 冷白。

当前 EXE 里发光经常和主体色冲突。马身、人物、光效、装饰各自变不同颜色，视觉上变成“彩虹噪声”。

### 3.4 材质有保护

AE 示例会保留：

- 皮肤不过度变色。
- 金属不过度染色。
- 白色毛发 / 布料保留明度。
- 黑色阴影不染荧光。
- 金边、装饰、武器有稳定色系。

当前 EXE 虽有 `skinSafeLayerKeys` 和 `shadowProtectThreshold`，但保护仍偏基础，缺少材质级随机策略。

## 四、当前 EXE 截图问题诊断

### 4.1 颜色太碎

很多方案同时出现：

- 青
- 绿
- 紫
- 粉
- 黄
- 蓝
- 红

这些颜色没有统一主题。单个小角色缩略图里颜色过多，高级感下降。

### 4.2 差异大，但品质不稳定

27 个方案确实有变化，但不是稳定好看的变化。

```text
变化度高 ≠ 设计感强
```

更好的目标：

```text
每个方案都有主题
每个主题内部统一
27 个方案之间互相有差异
```

当前状态：

```text
每个层随机
每个方案随机
整体不可控
```

### 4.3 发光色抢戏

角色本身较小，坐骑和特效发光面积大。一旦发光层随机到高饱和绿、粉、蓝，视觉中心会被发光吃掉。

优化方向：

```text
发光色可以鲜艳
但必须服务主色
不能独立乱跳
```

### 4.4 暗色方案不够高级

当前 `Dark` 档逻辑：

- 饱和度降。
- 亮度降。
- 对比度加。

如果每层仍独立随机 hue，暗色会变成“脏暗彩色”。

高级暗色需要：

- 黑 / 深蓝 / 深紫 / 深绿作为主体。
- 高光只留一小块亮色。
- 金属、边缘、发光给少量点缀。

不是简单压暗全部层。

### 4.5 随机过于平均

好看的角色皮肤不是所有部件同等变化。应有视觉层级：

| 视觉等级 | 部件 |
|---|---|
| 主视觉 | 衣服 / 坐骑主体 / 大面积布料 |
| 辅助视觉 | 武器 / 鞍具 / 飘带 / 装饰 |
| 点缀视觉 | 宝石 / 发光 / 魔法线条 |
| 保留视觉 | 皮肤 / 金属 / 阴影 / 白发 |

当前随机没有足够强的“角色部件权重”。

## 五、最重要优化方向

### 5.1 从“层随机”改成“方案级调色盘随机”

当前逻辑：

```text
每层 randomizeStack()
每层独立随机 hue/sat/lightness
```

建议改成：

```text
先为整个 scheme 生成 Palette
再按 layer 类型套用 palette
```

建议结构：

```cpp
struct SchemePalette {
    QColor primary;       // 主色
    QColor secondary;     // 辅色
    QColor accent;        // 点缀色
    QColor glow;          // 发光色
    QColor shadowTint;    // 阴影偏色
    QColor highlightTint; // 高光偏色
    int mood;             // 风格类型
};
```

推荐流程：

```text
随机方案风格
→ 生成主色
→ 按色彩和谐规则生成辅色 / 点缀 / 发光
→ 每个层根据语义选择颜色目标
→ 生成 EffectStack
→ 打分过滤
```

这是收益最大的改动。

### 5.2 加入色彩和谐模板

不要全范围乱抽 hue。使用固定配色关系。

| 模板 | 规则 | 适合 |
|---|---|---|
| 单色系 | 主色 ±15° | 高级、统一 |
| 类似色 | 主色 ±30° / ±45° | 自然、柔和 |
| 互补色 | 主色 + 180°，辅色低占比 | 强对比 |
| 分裂互补 | 主色 +150° / +210° | 游戏皮肤常用 |
| 三角色 | 主色 +120° / +240°，降低饱和 | 奇幻风 |
| 冷暖对比 | 冷主体 + 暖高光 | 高级感强 |
| 黑金 / 白金 | 低饱和主体 + 金色点缀 | 稀有皮肤感 |

示例：

```text
主色 = 紫
辅色 = 深蓝
点缀 = 金
发光 = 淡紫 / 冷白
```

比每层紫、绿、粉、蓝乱跳稳定很多。

### 5.3 限制每层 hue 偏移范围

当前：

```text
HSL hue: [-180, 180]
```

建议改成两级。

方案级主 hue：

```text
baseHue = random(0, 360)
```

层级 hue 偏移：

```text
主层：baseHue ± 8°
辅层：baseHue ± 20°
点缀层：accentHue ± 10°
发光层：glowHue ± 5°
```

推荐范围：

| 层类型 | hue 偏移 |
|---|---:|
| 主体衣服 / 坐骑身体 | ±8° 到 ±15° |
| 装饰 / 飘带 | ±20° 到 ±35° |
| 发光 / 魔法 | ±5° 到 ±15° |
| 金属 | 不改 hue 或仅 ±5° |
| 皮肤 | 跳过 |
| 阴影层 | 不改 hue，只调亮度 / 饱和度 |

### 5.4 随机所有层默认用统一调色盘

文档里当前有：

```text
randomizeAllLayers(sameSeedAllLayers)
UI 默认 false
```

这容易导致层间不协调。

建议：

```text
UI 默认改为 true
```

更好方案：新增智能随机路径：

- `随机当前层`
- `智能随机当前方案`
- `智能生成27套`
- `重抽点缀色`
- `锁定主色后重抽`

### 5.5 引入材质类型

不要只按 layer key 处理。给每个层增加材质标签。

```cpp
enum class MaterialRole {
    Skin,
    Hair,
    ClothMain,
    ClothSub,
    Metal,
    Leather,
    Glow,
    Weapon,
    MountBody,
    MountDecor,
    Shadow,
    Unknown
};
```

每种材质随机范围不同：

| 材质 | 随机策略 |
|---|---|
| Skin | 基本跳过，只允许轻微暖冷 |
| Hair | 可变，但保留明度 |
| ClothMain | 主色承载 |
| ClothSub | 辅色承载 |
| Metal | 低饱和，保亮度，少改 hue |
| Gold / Silver | 固定色池 |
| Glow | 高饱和，高亮度，但色相受 palette 控制 |
| Shadow | 保暗，不染荧光 |
| MountBody | 主色 / 低饱和大面积 |
| MountDecor | 点缀色 |

这比纯随机 `EffectStack` 更接近 AE 人工效果。

## 六、7 个效果的具体优化建议

### 6.1 HSL：从核心随机改为目标色校正

当前 HSL 承担太多随机任务。

不要：

```text
hue = random(-180, 180)
```

改成：

```text
hue = targetHue - sourceDominantHue + jitter
saturation = materialRange
lightness = preserveValueRange
```

先分析当前层原色主色，再变到目标色。红布变蓝布、蓝光变紫光会更稳定。

### 6.2 亮度/对比度：按风格统一

当前有些层亮、有些层暗，角色整体光照容易碎。

建议方案级 `ToneProfile`：

```cpp
struct ToneProfile {
    int globalBrightness;
    int globalContrast;
    int shadowLift;
    int highlightProtect;
};
```

推荐范围：

| 风格 | brightness | contrast |
|---|---:|---:|
| 写实 | -6 到 +4 | -4 到 +8 |
| 暗黑 | -18 到 -6 | +8 到 +18 |
| 明亮 | +4 到 +12 | -6 到 +6 |
| 稀有高光 | -4 到 +6 | +10 到 +22 |

### 6.3 曲线：从随机中点改为风格模板

曲线不应完全随机。建议固定模板：

| 模板 | 效果 |
|---|---|
| Soft | 稍微提中间调 |
| Contrast | 轻微 S 曲线 |
| Matte | 黑位抬高，低对比 |
| Deep | 压暗阴影 |
| GlowFriendly | 保高光，不压亮部 |

比随机中点 `(x, y)` 更可控。

### 6.4 通道混合器：继续默认关闭

当前随机不开通道混合器是正确判断。该效果容易产生怪异颜色，不适合普通写实随机。

建议只用于高级预设，不进入普通随机。

### 6.5 颜色平衡：方案级统一冷暖

当前颜色平衡每层三区主轴随机，容易导致阴影和高光偏色不统一。

建议整个 scheme 共用：

```text
shadowTint
highlightTint
```

经典搭配：

| 风格 | 阴影 | 高光 |
|---|---|---|
| 暗黑紫 | 蓝紫 | 暖金 |
| 冰霜 | 深蓝 | 青白 |
| 火焰 | 深红 | 橙黄 |
| 毒雾 | 深绿 | 黄绿 |
| 圣光 | 冷灰 | 金白 |
| 幻彩 | 紫蓝 | 粉白 |

### 6.6 照片滤镜：从每层随机改为全局氛围

照片滤镜适合做整体氛围，不适合每层乱开。

建议：

```text
每个 scheme 只选一个全局 PhotoFilter 倾向
```

示例：

- 冰霜方案：冷色 80 / 青。
- 火焰方案：暖色 85 / 橙红。
- 暗黑方案：深蓝 / 紫。
- 森林方案：翡翠绿 / 暖黄。
- 圣光方案：暖黄 / 金。

层内密度按材质缩放：

```text
主层 density 10-18
辅层 density 6-14
发光层 density 0-8
皮肤层 0-5
```

### 6.7 Vibrance：材质化处理

当前 `Vibrance` 在 Realistic / Dark 中常降饱和。对衣服可以，对发光不一定好。

建议：

| 材质 | vibrance | saturation |
|---|---:|---:|
| 布料 | -10 到 +8 | -15 到 +5 |
| 金属 | -20 到 -5 | -25 到 -10 |
| 发光 | +5 到 +25 | 0 到 +15 |
| 皮肤 | -5 到 +5 | -8 到 +4 |
| 阴影 | -30 到 -10 | -35 到 -15 |

## 八、质量评分系统

随机生成后，不要直接用。应生成多个候选，然后打分选最好。

推荐流程：

```text
每个方案生成 8-20 个候选
→ 快速渲染缩略图
→ 分析颜色质量
→ 保留最高分
```

### 8.1 色相数量惩罚

如果角色里主色超过 4 个，扣分。

```text
主色 <= 3 最好
主色 4 勉强
主色 >= 5 扣大分
```

### 8.2 饱和度惩罚

高饱和面积不能太大。

```text
高饱和像素面积 > 35% 扣分
高饱和像素面积 10%-25% 最好
```

### 8.3 明度对比检查

角色不能太灰，也不能过曝。

```text
暗部有细节
亮部不过曝
中间调占主体
```

### 8.4 发光占比检查

发光色面积太大，扣分。发光应是点缀，不是整身荧光。

### 8.5 原始结构保留

输出图与原图的亮度结构相似度要高。

可比较：

```text
source luminance histogram
output luminance histogram
```

明度结构差太多，扣分。

### 8.6 背景可读性

截图背景是深灰。角色不能整体也变成一团深灰。

检查：

```text
角色轮廓亮度 与 背景亮度 差值
```

差值太小，扣分。

## 九、推荐随机算法重构

### 9.1 当前算法

```text
for each scheme:
    for each layer:
        randomizeStack(layerEffect, seed=0)
```

### 9.2 推荐算法

```text
for each scheme:
    palette = generatePalette(style, schemeIndex)
    tone = generateToneProfile(style)

    for each layer:
        role = detectLayerRole(layer)
        target = palette.pick(role)
        stack = buildEffectStackFromTarget(layerSourceStats, target, role, tone)

    score = evaluateSchemePreview()
    if score too low:
        reroll
```

### 9.3 伪代码

```cpp
SchemePalette palette = generatePalette(style, seed);
ToneProfile tone = generateToneProfile(style, seed);

for (Layer& layer : project.layers) {
    if (!layer.visible || isSkinSafe(layer)) {
        continue;
    }

    MaterialRole role = classifyLayer(layer.key, layer.imageStats);
    LayerColorTarget target = palette.targetFor(role);

    EffectStack stack;
    stack.reset();

    applyHueTowardTarget(stack, layer.sourceDominantHue, target.hue, role);
    applySaturationByMaterial(stack, target.saturation, role);
    applyToneProfile(stack, tone, role);
    applyColorBalance(stack, palette.shadowTint, palette.highlightTint, role);
    applyPhotoFilter(stack, palette.atmosphereFilter, role);
    applyVibrance(stack, role);

    scheme.layerEffects[layer.key] = stack;
}
```

## 十、颜色空间建议

当前效果链在 sRGB / HSV 中工作，与 AE 对齐没有问题。

但“随机选颜色”不建议只用 HSV。建议调色盘生成使用 OKLCH 或 HSLuv 思路。

原因：

- HSV 的亮度感知不均匀。
- 同样 V=80，黄色会比蓝色亮很多。
- 随机出来蓝紫容易暗，黄绿容易刺眼。

如果不想引入复杂库，可先做简化规则：

```text
黄色 / 绿色：降低 saturation 和 value
蓝色 / 紫色：提高 lightness
红色 / 粉色：限制饱和面积
青色：限制发光强度
```

## 十一、优先级路线图

### P0：马上改，收益最大

1. `随机所有层` 改为方案级 `Palette`，不要每层独立全随机。
2. HSL `hue [-180,180]` 改成 `baseHue + layerOffset`。
3. 限制高饱和面积。`Vivid` 只给 Glow / Accent，不给全部主体层。
4. PhotoFilter 改成 scheme 级，一个方案一个氛围滤镜。
5. 颜色平衡阴影 / 高光统一，整个方案共享 `shadowTint` 与 `highlightTint`。

### P1：中期改，明显提升高级感

1. 增加材质角色，至少区分 `主体 / 装饰 / 金属 / 发光 / 皮肤 / 阴影`。
2. 预设 27 个风格槽。
3. 增加缩略图评分过滤。
4. 保留原始明度结构，所有换色优先保留 luminance。

### P2：后期高级优化

1. 自动分析每层主色：`dominant hue`、`average saturation`、`luminance range`、`alpha area`、`bright pixel ratio`、`dark pixel ratio`。
2. 自动识别发光层：高亮像素多、饱和度高、alpha 边缘软、面积小但亮度高。
3. 后续可用人工评分数据训练小型排序模型，但当前阶段规则评分已够用。

```text

主体 sat = [-35,-15]
主体 light = [-22,-8]

点缀 light = [+0,+10]

发光 sat = [+5,+25]

```


建议：

## 十五、最终建议

随机变色优化不要继续扩大随机范围。相反，要减少自由度，增加规则和层级。

最终方向：

```text
统一主色
统一冷暖
统一发光
材质分层
保留明度
候选评分
27 套风格化生成
```

最推荐先做 5 个改动：

1. `随机所有层` 改为方案级 `Palette`，不要每层独立全随机。
2. HSL `hue [-180,180]` 改成 `baseHue + layerOffset`。
3. 加 `MaterialRole`：至少区分 `主体 / 装饰 / 金属 / 发光 / 皮肤 / 阴影`。
4. 每个方案只用一个全局氛围滤镜和一组阴影 / 高光偏色。
5. 生成 27 套时按固定风格槽分布，不要 27 套同逻辑乱抽。

做完这 5 个，画面会从“随机染色”明显变成“皮肤配色”。
