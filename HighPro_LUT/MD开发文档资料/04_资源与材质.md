## 一、定义

**HighPro_LUT 没有传统意义上的"材质文件"** (无 `.mat` / `.shader` / `.material` 扩展名), 也**没有 Unity / Unreal 的 material asset 概念**.

这个工具的"材质"是**虚拟概念**, 由 4 个层次合成:

```
材质 (运行时) = (源 TGA 像素)
              + (HALD-CLUT 颜色映射 LUT)
              + (7 效果链参数 EffectStack)
              + (3 个 PS shader)
              + (固定 Pipeline State: Blend / Sampler / RTV)
```

每一层都是独立资源, 互相组合 → 最终上屏像素.

## 三、内置资源详解 (`assets/`)

### 3.1 `颜色图.png` — 恒等 HALD-CLUT

| 属性 | 值 |
|------|---|
| 格式 | PNG, 256×16, RGB (3 通道, 无 alpha) |
| 数据 | 16³ HALD-CLUT 平铺 |
| qrc alias | `:/lut/default.png` |
| 加载 | `SchemePanel::loadIdentityLut`, `HaldClut::load` |

**HALD-CLUT 编码** (颜色图.png 实证):

```
像素 (x, y), x∈[0,255], y∈[0,15]:
  R_in = (x mod 16) * 17        // x 内的 R 列
  G_in = y * 17                 // 行 = G
  B_in = (x / 16) * 17          // x 切片 = B
```

(实测 `颜色图.png[0,0] = (0,0,0)`, `[0,15] = (254,0,0)`, `[15,0] = (0,253,0)`, `[255,15] = (255,253,255)` 与上述编码完全吻合.)

**用途**:

- 启动时加载为 256×16 GPU 纹理, 绑到 `t1` (gLut)
- 用户拖效果滑块 → `LutBaker` 离屏 RT 烘焙 → 新 LUT (覆盖恒等 LUT)
- 用户保存方案 → `LutBaker` 烘焙 → 写盘 `add_lut/0N.png`

### 3.2 三个 HLSL — 实际"材质"

| 文件 | 文件大小 | 用途 | 何时使用 |
|------|---------|------|---------|
| `fullscreen_quad.hlsl` | ~2.7 KB | **透传** + Solid + Triangle | 加载 TGA 后, 没拖参数也没选 LUT |
| `recolor.hlsl` | ~3.1 KB | HALD-CLUT 三线性反推 | 选了某个方案 LUT, 但没拖效果链 |
| `effect_chain.hlsl` | ~9.3 KB | 7 效果串联 + 烘焙 | 拖动任意 EffectPanel 滑块 |

#### 3.2.1 `fullscreen_quad.hlsl`

**4 个 entry**:

| Entry | 类型 | 用途 |
|-------|------|------|
| `VSMain` | VS | 用 SV_VertexID 出 6 顶点全屏 quad, **被三个 hlsl 共用 entry 名** (各自独立编译) |
| `PSMain` | PS | 透传: 采样 t0 → sRGB→linear → 输出 (rgb, a) (Straight) |
| `PSMainSolid` | PS | 纯色填充 (M5 选中边框) |
| `PSMainTriangleUp` | PS | 实心 ▲ 三角 (M5 选中标识) |

**资源绑定**:

| Slot | 类型 | 含义 |
|------|------|------|
| `b0` | cbuffer `QuadCB` | uvRect (4) + posRect (4) + tint (4), 共 48 B |
| `t0` | Texture2D `gTex` | 当前帧 TGA (R8G8B8A8_UNORM, Straight Alpha) |
| `s0` | SamplerState `gLinearClamp` | LINEAR/POINT 二选一 (FrameRenderer 决定), CLAMP |

**代码**:

```hlsl
float4 PSMain(VSOut i) : SV_TARGET
{
    float4 c = gTex.Sample(gLinearClamp, i.uv);
    if (c.a < 0.001) return float4(0, 0, 0, 0);
    c.rgb = srgbToLinear(saturate(c.rgb));
    return c * tint;     // straight, 不预乘
}
```

#### 3.2.2 `recolor.hlsl`

**2 个 entry**: `VSMain` (同 fullscreen_quad), `PSMain` (HALD-CLUT 三线性反推).

**资源绑定**:

| Slot | 类型 | 含义 |
|------|------|------|
| `b0` | cbuffer `QuadCB` | (与 fullscreen_quad 相同) |
| `t0` | Texture2D `gFrame` | 源 TGA |
| `t1` | Texture2D `gLut` | HALD-CLUT 256×16 (烘焙后或恒等) |
| `s0` | SamplerState `gFrameSampler` | POINT/LINEAR + CLAMP |
| `s1` | SamplerState `gLutSampler` | LINEAR + CLAMP (LUT 三线性必须 LINEAR) |

**核心算法** (HALD 16³):

```hlsl
float3 SampleHald(float3 c)   // c ∈ [0,1] sRGB 直通色
{
    float3 q = saturate(c) * 15.0f;
    float r = q.r, g = q.g, b = q.b;
    float b0f = floor(b);
    float b1f = min(b0f + 1.0f, 15.0f);
    float bf  = b - b0f;
    float u0 = (b0f * 16.0f + r + 0.5f) / 256.0f;
    float u1 = (b1f * 16.0f + r + 0.5f) / 256.0f;
    float v  = (g + 0.5f) / 16.0f;
    float3 c0 = gLut.SampleLevel(gLutSampler, float2(u0, v), 0).rgb;
    float3 c1 = gLut.SampleLevel(gLutSampler, float2(u1, v), 0).rgb;
    return lerp(c0, c1, bf);     // B 切片间手动线性插值; R/G 由 LINEAR sampler 自动双线性
}
```

#### 3.2.3 `effect_chain.hlsl`

**3 个 entry**:

| Entry | 类型 | 用途 |
|-------|------|------|
| `VSMain` | VS | 同 fullscreen_quad |
| `PSMain` | PS | 7 效果串联 (运行时预览) |
| `PSMainBake` | PS | 离屏烘焙 256×16 LUT, 写 `add_lut/0N.png` |

**资源绑定**:

| Slot | 类型 | 含义 |
|------|------|------|
| `b0` | cbuffer `QuadCB` | (同上) |
| `b1` | cbuffer `EffectCB` | 7 效果参数, 见下表 |
| `t0` | Texture2D `gFrame` | 源 TGA (烘焙时是颜色图.png) |
| `t1` | Texture2D `gCurve` | 256×1 曲线 LUT (RGBA = master/r/g/b) |
| `s0` | SamplerState `gFrameSampler` | POINT/LINEAR + CLAMP |
| `s1` | SamplerState `gCurveSampler` | LINEAR + CLAMP |

**`EffectCB` 布局** (~120 B, 与 C++ `ColorEffect.h` 严格对应):

```
enableMask : int4    x=bitmask(7位), y=shadow_protect(0..255), zw 备
hsl        : float4  x=hue/360, y=sat/100, z=lit/100, w 备
brtCtr     : float4  x=brt/255, y=ctr/100, z 备 w 备
mixR/G/B   : float4×3  通道混合矩阵 + 常数项
mixFlag    : int4    x=monochrome
balShadow  : float4  影部 R/G/B
balMid     : float4  中调 R/G/B
balHigh    : float4  亮部 R/G/B
balFlag    : int4    x=preserveLuma
photoColor : float4  xyz=滤镜色, w=density
photoFlag  : int4    x=preserveLuma
vibrance   : float4  x=vibrance, y=saturation
```

**7 效果顺序** (`applyEffectChain`):

1. HSL (Hue / Saturation / Lightness)
2. Brightness / Contrast
3. Curves (master + per-channel, 256-LUT 采样 t1)
4. Channel Mixer (3×3 + 常数, monochrome 选项)
5. Color Balance (影/中/亮三段 R/G/B + preserveLuma)
6. Photo Filter (颜色 + 密度, preserveLuma)
7. Vibrance (vibrance + saturation)

完成后 `srgbToLinear` 写 sRGB RTV.

## 四、`assets.qrc` 资源清单

```xml
<RCC version="1.0">
    <qresource prefix="/">
        <file alias="lut/default.png">颜色图.png</file>
        <file alias="shaders/fullscreen_quad.hlsl">shaders/fullscreen_quad.hlsl</file>
        <file alias="shaders/recolor.hlsl">shaders/recolor.hlsl</file>
        <file alias="shaders/effect_chain.hlsl">shaders/effect_chain.hlsl</file>
    </qresource>
</RCC>
```

CMake 通过 `qt_add_resources(APP_RESOURCES "assets/assets.qrc")` 把这些文件**编译进 EXE** (生成 `qrc_assets.cpp` ≈ 100 KB).  
代码侧用 `QFile(":/shaders/recolor.hlsl").readAll()` 读取, **运行时不需要 `assets/` 目录**.

## 六、Pipeline State (固定材质属性)

`FrameRenderer::initialize` 一次创建, 所有渲染共用:

| State | 配置 | 备注 |
|-------|------|------|
| BlendState | `SRC_ALPHA / INV_SRC_ALPHA` | Straight Alpha over |
| Rasterizer | `FillSolid + CullNone + DepthClip` | 全屏 quad |
| DepthStencil | disabled | 2D 不要深度 |
| InputLayout | `nullptr` | 用 SV_VertexID, 无 IA buffer |
| Topology | `TRIANGLELIST` | 6 顶点 = 2 三角形 |
| Sampler (frame) | `LINEAR + CLAMP` / `POINT + CLAMP` | 1:1 时 POINT, 缩放时 LINEAR |
| Sampler (lut) | `LINEAR + CLAMP` | HALD 三线性必须 LINEAR |
| RTV | `B8G8R8A8_UNORM_SRGB` | GPU 自动 linear → sRGB encode |

详见 `03_TGA加载与渲染管线.md`.

## 七、Shader 路径分流逻辑

`FrameRenderer.cpp` L402-403 每帧每层选一条:

```cpp
useEffect = (有 EffectStack && !isIdentity)         // 拖了任意效果滑块
useLut    = (无 effect && lut.isValid())            // 选了方案 LUT 但没拖效果

if      (useEffect)  → effect_chain.hlsl  PSMain
else if (useLut)     → recolor.hlsl       PSMain
else                 → fullscreen_quad.hlsl PSMain   ← 纯导入 TGA 显示
```

| 用户操作 | 走的 hlsl |
|---------|-----------|
| 加载 9353 资源, 不动 | `fullscreen_quad.hlsl` |
| Ctrl+1..6 切换方案 LUT | `recolor.hlsl` |
| 拖 EffectPanel 任意滑块 | `effect_chain.hlsl` |
| Ctrl+E 烘焙导出 | `effect_chain.hlsl::PSMainBake` |

## 八、新增 / 修改 Shader 流程

### 8.1 改现有 hlsl

1. 编辑 `assets/shaders/xxx.hlsl`
2. **不需重 configure**, 直接 `cmake --build build\release`
3. AUTORCC 检测到 qrc 内引用文件改动 → 重 generate `qrc_assets.cpp` → 链 EXE
4. EXE 启动时通过 `D3DCompile` 运行时编译 hlsl, 即时生效

### 8.2 加新 hlsl

1. 在 `assets/shaders/` 新建 `.hlsl`
2. **改 `assets/assets.qrc`**, 加 `<file alias="shaders/xxx.hlsl">shaders/xxx.hlsl</file>`
3. 在 `FrameRenderer.cpp` 加 `m_shaderXxx.compile(..., "VSMain", "PSMain", "xxx", errorOut)` 编译入口
4. 加分支判断 → `dc->VSSetShader/PSSetShader/...`
5. 加新 cbuffer? 同步加 `D3D11_BUFFER_DESC` + `Map/Unmap`
6. 加新 SRV? 同步在 SamplerState / PSSetShaderResources 处理

### 8.3 改 cbuffer 布局

C++ 端 `ColorEffect.h` (CB 镜像) **必须**与 hlsl `cbuffer EffectCB` 字段一一对应:

```
hlsl  cbuffer EffectCB { ... };
        ↕  字段顺序 / 类型 / packing 全部相同
C++   struct EffectCB  { ... };       (POD, alignas(16))
```

不一致 → GPU 拿到错值, 但**不会编译报错**, 只表现为渲染异常. 修改时务必 grep `EffectCB` 双向核对.

## 九、为何不用 .mat / .shader 资产

| 维度 | HighPro_LUT | Unity / Unreal Material |
|------|-------------|------------------------|
| 目标 | 离线 LUT 烘焙工具, 一类资源固定流程 | 通用游戏引擎, 千变万化 |
| Shader 数 | 3 个固定 hlsl | 用户动态创建 |
| 状态切换 | 编译期固定 + 运行期 if/else 选 PS | Material Instance / Variant |
| 资产管理器 | 不需要 — qrc 编译进 EXE | Asset Database + GUID + Inspector |
| 用户配置 | EffectStack 参数 (滑块值) | Material Asset 内嵌纹理/参数 |
| 跨工程复用 | 用户存 .json 工程文件 | .mat / .uasset 资产 |

简言之: **HighPro_LUT 是单一管线工具, "材质" = 3 个固定 PS + 用户参数, 不需要资产抽象层.**  
若未来扩展多种渲染 (例如加 Bloom / Glow), 才考虑引入 PSO + Material 抽象.
