# HighPro_LUT 使用指南

> 版本: **v1.0.1**
> 适用 EXE: `HighPro_LUT.exe` (动态 / Qt dll 同目录) · `LUT_Pro.exe` (静态单 EXE)
> 安装包: `HighPro_LUT_setup.exe` (7z SFX 自解压到 `%TEMP%` 启动)
> 适用系统: Windows 10 / 11 (x64), 显卡支持 Direct3D 11 Feature Level 11.0+

---

## 一、工具是什么

**导入角色 TGA 序列帧 → 多层合成实时预览 → 用 7 个 AE 风格颜色效果给每层调色 → 烘焙出多套换色方案 (add_lut/0N.png) → 批量输出 PNG 颜色贴图给游戏引擎用。**

一句话:
- **输入**: 一套角色的 TGA 序列帧 (按 `body / 00..04 / addon` 分层, 每层下面分动作目录)
- **输出**: 每层 `add_lut/0N.png` (256×16 HALD-CLUT) + 可选 GIF 预览图

特色:
- GPU 烘焙 + GPU 变色, 拖滑块全程 60FPS 顺滑
- 一套素材一次最多保存 28 套配色方案, 画廊缩略图实时刷新
- 影子像素天然免疫变色 (HALD-CLUT 0 角点 + 影子保护阈值双保险)
- 智能随机按 LayerSlot 半语义分块, 一键出一整套统一配色

## 二、安装与启动

### 2.1 三种发布形态

| 形态 | 文件 | 是否需 Qt dll | 双击运行 | 适合场景 |
|------|------|----------------|----------|----------|
| 动态 EXE | `HighPro_LUT.exe` + 同目录 Qt6\*.dll + 插件目录 | 是 | 是 | 开发期 / 调试 |
| 静态单 EXE | `LUT_Pro.exe` | 否 | 是 | 拷给同事 / 绿色版 |

- 双击 EXE → 启动空白主窗口
- 标题栏: `HighPro_LUT - 高性能序列帧角色换色工具`
- 上次正常退出过则会自动加载上次的 `.hplut.json` 工程 (可在设置里关掉自动恢复)

### 2.3 系统要求

- Windows 10 / 11 64 位
- D3D11 设备 (集显也可, 烘焙仅 256×16)
- 路径支持中文 (PathUtil 走宽字符, windeployqt 中转例外)

## 四、第一次上手 (5 分钟跑通)

### Step 1 · 打开素材

`文件 → 打开源目录...` (快捷键 `Ctrl+O`), 选含分层的角色根目录, 如:

```
9353/
├── body/
│   ├── stand/
│   │   ├── 0000.tga      ← 方向 0 帧 000
│   │   ├── 0001.tga      ← 方向 0 帧 001
│   │   └── ...
│   ├── walk/
│   ├── attack/
│   └── add_lut/          ← 可选: 老的烘焙 PNG (Ctrl+1..6 加载)
├── 00/
├── 01/
├── 02/
├── 03/
├── 04/
└── addon/
    ├── 01/
    └── 02/
```

帧名约定: `^(\d)(\d{3})\.tga$` (第 1 位 = 方向 0..7, 后 3 位 = 帧序号 000..)

扫描成功 → `资源树` 出现层列表, `预览画布` 开始播放 stand 动作方向 0。

### Step 2 · 看本体

- 默认进入 `方案 0 - 默认本体`, 所有效果禁用, 显示原始 TGA
- 本体方案 **不可编辑** — 颜色效果面板会提示 "本体方案不可编辑"

### Step 3 · 新增方案

两种方式:
1. `方案 → 新增方案` (`Ctrl+N`)
2. 方案画廊右键 → `新增方案`

最多 28 个方案 (0 本体 + 1..27 用户方案)。新方案默认就是本体的拷贝, 各层全 disabled。

### Step 4 · 调色

- 在 `资源树` 点某一层 (如 `00`) → `颜色效果` 顶部显示 "当前层: 00"
- 勾选 `1. 色相/饱和度` → 拖动色相滑块 → 画布实时变色
- 7 个效果各自独立开关 + 滑块, 全部数字与 Adobe AE 对齐

### Step 5 · 智能一键出方案

`颜色效果` 底部按钮区:

| 行 | 按钮 | 作用 |
|---|------|------|
| 0 | 🎨🎲 智能+随机 | 所有非本体方案: 50% 智能值 + 50% 旧随机值的混合 |
| 1 | 🎨 智能当前层 / 智能所有层 / 智能可编辑 / 智能全部 | 按 LayerSlot 智能随机, 整套方案色相统一 |
| 2 | 🎲 随机当前层 / 随机所有层 / 随机可编辑 / 随机全部 | 旧版纯随机 (每层各自乱来) |
| 3 | ↺ 重置当前层 / 重置所有层 / 重置可编辑 / 重置全部 | 把效果栈清零 |

**推荐工作流**:
1. `方案 → 新增方案` × N 次, 准备好 N 个空白方案槽
2. `颜色效果 → 🎨 智能可编辑` 一键给所有可编辑方案上色
3. 方案画廊选个不满意的 → 右键 `❤️细化方案` 微调 HSL/亮度对比度
4. 满意的方案右键点击 🔒 锁定 (避免后续被覆盖)
5. `导出 → 导出已锁定的全部方案 add_lut PNG...` (`Ctrl+Alt+E`)

### Step 6 · 导出 LUT

`导出` 菜单 3 选 1:

| 入口 | 快捷键 | 范围 |
|------|--------|------|
| 导出当前方案 add_lut PNG... | `Ctrl+E` | 仅当前选中方案 |
| 导出全部方案 add_lut PNG... | `Ctrl+Shift+E` | 所有非本体方案 |
| 导出已锁定的全部方案 add_lut PNG... | `Ctrl+Alt+E` | 仅 🔒 锁定方案 |

弹窗选输出根目录 → 每层在该目录下自动建 `<层名>/add_lut/0N.png`。

### Step 7 · 保存工程

`文件 → 保存工程` (`Ctrl+S`) → `*.hplut.json`, 含:
- 所有方案的 EffectStack 参数
- LayerSlot 半语义分配
- 肤色保护层集合
- 锁定状态 / 当前编辑方案 / 窗口几何

下次双击 EXE 自动加载, 也可 `文件 → 最近工程` 选历史工程。

## 六、核心面板详解

### 6.1 资源树 (左 dock)

**UI 视角顺序** (AE 风格, 倒序): `addon → 00..04 → body`
**实际合成顺序** (Project::layers): `body → 00..04 → addon` (自下而上 SourceOver)

行为:
- **body / 00..04**: checkbox 独立显隐
- **addon (单选)**: 选中 = 显示该子层, 已选项再点 = 取消显示 (无 addon)
- **拖动 checkbox 一行**: 拖过的层 checkbox 同步翻转 (快速批量显隐)
- **右键菜单**:
  - 🛡️ 设为 / 取消肤色保护层 (双向同步 LayerSlot::Skin)
  - 🎭 设置 LayerSlot (8 种半语义分类)
  - 📋 复制层路径

每个层名前会自动加 emoji 前缀, 直观反映 slot:
- 🙋 Skin · 🦱 Hair · 👕 Clothing · 👗 Skirt · 💎 Decor01 · ✨ Decor02 · ⚔️ WeaponMetal · 🪵 WeaponNonMetal · ❓ Unknown

### 6.2 预览画布 (中央)

画布上面是 D3D11 实时渲染, 下面是控制条 (3 行):

**第 1 行**: 动作下拉 + 方向按钮 1..8 + 帧率 SpinBox + ⏯ 播放暂停 + 当前帧 `[5/12]`

**第 2 行**: 背景色 + 背景图 + 清除背景图 + 全屏画布按钮 + 隐藏未锁定方案 toggle + 缩放 HUD

**第 3 行**: 字符 X/Y 间距 (拼接画布时的负间距, 默认 -300 / -260) + 方案 ID label 开关 + Label Y 间距 + GIF 导出区

**多方案拼接**: 画布按 7 列 × N 行铺所有方案 (含本体), 每格画对应方案下的合成结果。`隐藏未锁定方案` 按钮可只看锁定方案。

**输出 GIF**:
- 循环次数: 0 = 无限循环, N = 播 N 次
- GIF ID: 是否在 GIF 上画方案编号 label
- 一键导出: 全局调色板 Floyd-Steinberg dither, 输出 `<源目录名>_<动作>_<N>frame.gif`

### 6.3 颜色效果 (底 dock)

7 个分组 (与 AE 标准链完全对齐), 每个分组顶部有 checkbox 总开关:

| # | 名称 | 关键参数 | 范围 |
|---|------|---------|------|
| 1 | 色相/饱和度 | 色相 / 饱和度 / 亮度 | -180~180 / -100~100 / -100~100 |
| 2 | 亮度/对比度 | 亮度 / 对比度 | -150~150 / -100~100 |
| 3 | 曲线 | Master + R + G + B 各 8 点, 256×220 自绘编辑器 | 0~255 |
| 4 | 通道混合器 | 3×4 仿射 (RR/RG/RB/常量 × 3 行) + 单色 toggle | -200~200% |
| 5 | 颜色平衡 | 阴影 / 中间调 / 高光 各 R/G/B, 保亮度 toggle | -100~100 |
| 6 | 照片滤镜 | 18 种 AE 预设 + 自定义 RGB + 密度 + 保亮度 | density 0~100% |
| 7 | 自然饱和度 | 自然饱和度 / 普通饱和度 | -100~100 / -100~100 |

**底部按钮区** (4 行, 详见上面 Step 5 表):
- 智能 = 走 LayerSlot + SchemePalette 配色逻辑
- 随机 = 旧版每层独立乱来
- 重置 = 清零回默认
- 智能+随机 = 50/50 混合

底部还会显示当前方案的 SchemePalette 详情 (主色相 / 辅助色 / 点缀 / 风格氛围)。

### 6.4 方案画廊 (右 dock)

每行 = 64×80 缩略图 + 方案名 + 🔒 lock 图标 (锁定后导出走这条) + 标记 emoji。

操作:
- 左键: 切换当前方案 (`Ctrl+]` / `Ctrl+[` 等价)
- 点击空白: 取消选中 (currentSchemeIndex = -1)
- 双击: 重命名
- 右键: **复制方案 / 删除方案 / 重命名 / 🔒 锁定 toggle / ❤️细化方案 / 🔷配色方案转移**

**❤️细化方案**:
- 打开独立弹窗, 左侧层列表 + 右侧仅 HSL/亮度对比度 两个简化效果
- 拖动滑块实时预览 + debounce 100ms 写回, 不动其他 5 个效果
- 弹窗内有独立 undo / redo (`Ctrl+Z` / `Ctrl+Y`)
- 已烘焙方案打开时弹确认: 会降级为可编辑, 原 add_lut PNG 引用清空

**🔷配色方案转移**:
- 两级子菜单: `按 LayerSlot 分组 → 目标方案 ID`
- 把源方案某一组 (按 Clothing/Hair/Decor 等 slot) 的 EffectStack 复制到目标方案对应层
- 用于跨方案借色

缩略图烘焙在后台线程异步刷新 (ThumbnailWorker), 不阻塞主线。

## 七、典型工作流

### 工作流 A · 全自动出一套 20+ 配色

```
1. Ctrl+O 选素材
2. 方案 → 新增方案 × 20 (Ctrl+N × 20)
3. 资源树右键, 把头发层标 Hair, 装饰层标 Decor01/02
   (其他层走 LayerKind 启发式, 默认就够)
4. 颜色效果 → 🎨 智能可编辑
5. 方案画廊翻一遍, 满意的右键 → 🔒 锁定
6. 不满意的 → 🎨 智能当前层 单独重摇
7. Ctrl+Alt+E 导出已锁定方案
8. Ctrl+S 保存工程, 下次还能继续微调
```

### 工作流 B · 手工细调一套高质量配色

```
1. Ctrl+O 选素材
2. 方案 → 新增方案 (Ctrl+N)
3. 资源树选 00 (服装层) → 颜色效果勾选色相/饱和度 + 亮度对比度 + 照片滤镜
4. 拖滑块直到画布显示满意效果
5. 资源树切 01 → 类似调
6. 全部层调完 → 方案画廊重命名为 "暗夜紫" 之类
7. 🔒 锁定
8. Ctrl+E 单方案导出
```

### 工作流 C · 复用老的 add_lut/0N.png

```
1. 素材里已经存在 9353/body/add_lut/01.png ~ 06.png
2. 打开素材后, Ctrl+1..6 把对应 LUT 应用到所有层
3. 画布立刻显示老烘焙效果 (走 recolor.hlsl, 不走 effect_chain)
4. 想改 → 方案画廊右键 → ❤️细化方案, 弹确认 "会降级为可编辑"
5. 调完重新导出, 覆盖原 PNG
```

### 工作流 D · 输出展示 GIF

```
1. 加载素材后, 拼好 28 个方案
2. 选好动作 (stand / walk) + 方向
3. 预览画布右下设置:
   - 循环: 0 (无限)
   - GIF ID: 勾选 (画上方案编号)
4. 点 "输出 GIF 图..." → 选保存路径
5. 等若干秒 (取决于帧数 × 方案数), 输出多方案拼接 GIF
```

## 八、Q & A

### Q1 · 扫描后某层没出现?

- 检查该层目录下至少要有 1 个动作子目录, 子目录里要有 `\d{4}\.tga` 帧
- 帧名第 1 位 = 方向 (0..7), 不符合规则的整体跳过

### Q2 · 智能随机出的颜色看起来很乱?

- 检查 LayerSlot 是否正确分配 (资源树右键 → 设置 LayerSlot)
- 默认启发式: `body→Skin`, `num_00→Clothing`, `01→Skirt`, `02→Hair`, `03→Decor01`, `04→Decor02`
- 角色穿装与默认不符时一定要手工标 slot, 否则色相分配会错位

### Q3 · 颜色效果面板灰掉, 提示 "本体方案不可编辑"?

- 当前选中的是 `方案 0 - 默认本体`, 不能改, 是设计
- 先 `Ctrl+N` 新增方案, 再调

### Q4 · 拖滑块时画布卡顿?

- 单 LUT 烘焙 < 1ms, 单帧变色 < 1ms, 极少卡
- 帧 LRU 池默认 256 帧, 4000+ 帧素材首次播放会预热, 之后顺滑
- 若仍卡, 关掉方案画廊 (减少缩略图烘焙)

### Q5 · 影子部分被染了颜色?

- HALD-CLUT (0,0,0) 角点天然不变, 效果链对 luma < 阈值 (默认 8/255) 也会渐变保护
- 若仍染色, 检查照片滤镜或颜色平衡的 "保亮度" 是否关掉
- 调试: `F12` 抓桌面 our.png + reference.png 对照

### Q6 · 工程文件路径含中文打不开?

- 已通过, `PathUtil` 走宽字符 API, 路径含中文/特殊字符均 OK
- 例外: `windeployqt` 部署阶段不支持中文路径, build.bat 自动用 `%TEMP%` 中转 (开发用)

### Q7 · 导出的 PNG 拿到游戏里色差?

- 工具默认 Gamma (sRGB 字节) 空间合成, 与 AE 默认 / Unity URP / Unreal "线性合成关" 行为一致
- 若游戏走 Linear 合成, 烘焙 PNG 后游戏要再做 sRGB→Linear → 颜色会浅
- 解决: 在游戏里把变色贴图标记为 sRGB, 引擎自动转换

### Q8 · 想批量给同一角色另一套素材应用相同方案?

- 工程文件 `.hplut.json` 记录的是 `layerKey → EffectStack`, layerKey 与素材根目录无关
- 另一套素材层名一致时, 直接 `打开工程` → `扫描 → 重新扫描`, 方案自动复用

### 9.1 注册表 (`HKCU\SOFTWARE\HighPro\HighPro_LUT`)

- 实际 TGA 帧 (按路径引用, 不嵌入)
- D3D 纹理 / 帧缓存 (运行期 LRU, 不持久化)
- 缩略图缓存 (运行期内存, 切方案后重烘)
