## 1. 总览

`HighPro_LUT` 是 Windows 桌面序列帧角色换色工具。核心逻辑不是直接替换像素颜色，而是为每个方案、每个图层维护一套 `EffectStack`，再通过 GPU 端 7 效果链实时渲染或离屏烘焙成 `add_lut` PNG。

当前颜色随机体系分三类:

| 类型 | 入口 | 核心函数 | 特点 |
|---|---|---|---|
| 旧随机 | `随机当前层` / `随机所有层` / `随机可编辑` / `随机全部` | `randomizeStack` | 每层独立生成效果参数，偏参数随机。 |
| 智能随机 | `智能当前层` / `智能所有层` / `智能可编辑` / `智能全部` | `generatePalette` + `randomizeStackBySlot` | 先生成方案级调色盘，再按 `LayerSlot` 分层应用。 |
| 智能+随机 | `智能+随机` | `blendStack` | 智能结果与旧随机结果按 50/50 混合。 |

当前推荐使用 **智能随机** 或 **智能+随机**，因为它能保证同一方案内服装、裙摆、装饰、发光、武器颜色有统一主题。

## 4. 效果链模型

### 4.1 `EffectStack`

每个方案的每个层都有一份 `EffectStack`。它包含 7 个效果开关和参数:

```cpp
std::array<bool, 7> enabled;
HslParams         hsl;
BrtCtrParams      brtCtr;
CurveParams       curves;
ChMixParams       chMix;
ColorBalParams    colorBal;
PhotoFilterParams photoFilter;
VibranceParams    vibrance;
int shadowProtectThreshold = 8;
```

7 效果顺序固定:

| 位 | 效果 | 结构 | 说明 |
|---:|---|---|---|
| 0 | 色相/饱和度 | `HslParams` | hue、saturation、lightness。 |
| 1 | 亮度/对比度 | `BrtCtrParams` | AE 经典亮度对比公式。 |
| 2 | 曲线 | `CurveParams` | Master + R/G/B 四条曲线。 |
| 3 | 通道混合器 | `ChMixParams` | 3×4 通道矩阵。随机默认不开。 |
| 4 | 颜色平衡 | `ColorBalParams` | 阴影/中间调/高光三区调色。 |
| 5 | 照片滤镜 | `PhotoFilterParams` | AE 风格滤镜色 + 浓度。 |
| 6 | 自然饱和度 | `VibranceParams` | 对低饱和区域更敏感。 |

额外机制:

| 名称 | 字段 | 默认 | 作用 |
|---|---|---:|---|
| 影子保护 | `shadowProtectThreshold` | 8 | 源像素低亮度时插值回原色，防止黑边/暗部被染脏。 |

### 4.2 HSL

| 参数 | 范围 | 说明 |
|---|---:|---|
| `hue` | -180 到 180 | 色相旋转角度。 |
| `saturation` | -100 到 100 | -100 灰阶，+100 约等于 2 倍饱和。 |
| `lightness` | -100 到 100 | HSV/HSB 中亮度方向调整。 |

项目使用 HSV/HSB 风格逻辑，与 AE 行为更接近。

### 4.3 亮度/对比度

| 参数 | 范围 | 说明 |
|---|---:|---|
| `brightness` | -150 到 150 | shader 内除以 255。 |
| `contrast` | -100 到 100 | shader 内除以 100。 |

公式:

```text
output = (input - 0.5) * (1.0 + contrast) + 0.5 + brightness
```

### 4.4 曲线

`CurveParams` 包含:

```text
master
r
g
b
```

每条曲线控制点范围 `0..255`。默认 `{(0,0), (255,255)}`。

曲线由 `CurveSolver` 编译成 256 点 LUT，再上传为 256×1 的 `R8G8B8A8_UNORM` 纹理。

当前智能随机使用多种固定曲线模板:

| 模板 | 用途 |
|---|---|
| `DeepSafe` | 深色服装，压暗暗部但保高光。 |
| `HighlightSafe` | 亮色服装，提亮中高光。 |
| `SoftContrast` | 中间调，轻微 S 曲线。 |
| `UltraDark` | 极暗档，黑位更低。 |
| `UltraBright` | 极亮档，亮但防过曝。 |

### 4.5 通道混合器

默认单位阵:

```text
R' = 100%R
G' = 100%G
B' = 100%B
```

旧随机与智能随机都默认不开通道混合器。原因是容易产生怪异颜色，不适合批量生成写实游戏角色方案。

### 4.6 颜色平衡

分三区:

| 区域 | 字段 |
|---|---|
| 阴影 | `sR/sG/sB` |
| 中间调 | `mR/mG/mB` |
| 高光 | `hR/hG/hB` |

默认 `preserveLuma = true`，防止明度结构被破坏。

智能随机中，`ClothingTone` 会决定颜色平衡方向，例如暗蓝、暗红、暗紫、暗绿、青黑、黑灰、暗棕等。

## 5. 旧随机系统

### 5.1 核心函数

```cpp
void randomizeStack(EffectStack& s, quint32 seed = 0);
```

位置:

```text
src/core/ColorEffect.cpp
```

特点:

1. 用 `std::mt19937`。
2. `seed != 0` 时固定随机种子。
3. `seed == 0` 时使用 `QRandomGenerator::global()->generate()`。
4. 先保存 `shadowProtectThreshold`，`reset()` 后恢复。
5. HSL 必开，其余效果按概率开。

### 5.2 风格档位

| 档位 | 概率 | 特点 |
|---|---:|---|
| `Realistic` | 70% | 写实，低饱和，接近原色。 |
| `Dark` | 20% | 压暗，大幅脱色。 |
| `Natural` | 8% | 自然增强，轻微饱和。 |
| `Vivid` | 2% | 高饱和创意色，低概率彩蛋。 |

## 6. 旧随机的问题

旧随机底层效果链完整，但它是“每层参数随机”，不是“方案设计随机”。典型问题:

1. 每层 hue 都在 `-180..180` 全范围独立随机。
2. 同一方案内服装、裙摆、头发、装饰、发光可能完全不统一。
3. 发光层容易抢戏。
4. 暗色方案可能变成脏暗彩色。
5. 颜色数量过多，缺少主色/辅色/点缀色层级。
6. 皮肤、金属、阴影只有基础保护，缺少材质级策略。

因此新增了 P0 智能随机路径。

## 7. 智能随机系统

### 7.1 核心思想

智能随机从“层独立随机”改为:

```text
先生成 SchemePalette
→ 再按 LayerSlot 给每层分配角色
→ 每层按角色生成 EffectStack
```

核心函数:

```cpp
SchemePalette generatePalette(int schemeIdx, quint32 seed = 0);
void randomizeStackBySlot(EffectStack& s, LayerSlot slot, const SchemePalette& palette, quint32 seed = 0);
```

### 7.2 `SchemePalette`

```cpp
struct SchemePalette {
    int          primaryHue;
    int          secondaryHue;
    int          accentHue;
    int          accent2Hue;
    int          glowHue;
    MetalTone    metal;
    StyleMood    mood;
    ClothingTone clothing;
    int          saturationBias;
    int          lightnessBias;
};
```

字段含义:

| 字段 | 含义 |
|---|---|
| `primaryHue` | 主色，通常给服装主体。 |
| `secondaryHue` | 辅色，通常给裙摆/头发。 |
| `accentHue` | 主点缀色，通常给装饰 01。 |
| `accent2Hue` | 高亮点缀，通常给装饰 02。 |
| `glowHue` | 发光备用色。 |
| `metal` | 金属色池。 |
| `mood` | 方案氛围。 |
| `clothing` | 服装主色调，用于颜色平衡/照片滤镜方向。 |
| `saturationBias` | 编码后的饱和度档位 + jitter。 |
| `lightnessBias` | 编码后的亮度档位 + jitter。 |

### 10.1 LayerSlot 定义

```cpp
enum class LayerSlot : int {
    Unknown = 0,

    Hair,
    Clothing,
    Skirt,
    Decor01,
    Decor02,
    WeaponMetal,
    WeaponNonMetal,

```

含义:

| Slot | 中文 | 智能随机职责 |
|---|---|---|
| `Unknown` | 未指定 | 兜底按 Clothing。 |
| `Skin` | 肤色 | 不变色，保护。 |

| `WeaponMetal` | 武器金属 | 固定金属色池。 |
| `WeaponNonMetal` | 武器非金属 | 跟服装或装饰统一。 |

### 10.2 默认 slot 启发式

如果用户没手动指定 `layerSlots`，项目按层类型推断:

| 层类型 | 默认 Slot |
|---|---|
| `body` | `Skin` |
| 数字层 `00` | `Clothing` |
| 数字层 `01` | `Skirt` |
| 数字层 `02` | `Hair` |
| 数字层 `03` | `Decor01` |
| 数字层 `04` | `Decor02` |
| 数字层 `05+` | `Decor01` |

优先级:

```text
skinSafeLayerKeys 命中 → Skin

```

### 10.3 肤色保护

`skinSafeLayerKeys` 是最终保护源。命中后:

1. 随机操作跳过该层。
2. `slotFor()` 永远返回 `Skin`。
3. `randomizeStackBySlot(Skin)` 会 reset 到全效果关闭。

## 11. 智能随机各 Slot 策略

### 11.1 Skin

```text
reset EffectStack
所有 effect disabled
保留 shadowProtectThreshold
```

结果: 完全不变色。

### 11.2 Hair

目标:

```text
secondaryHue ±18°
```

当前策略:

| 分支 | 概率 | saturation | lightness |
|---|---:|---:|---:|
| 暗发 | 30% | -15 到 0 | -12 到 -2 |
| 普通 | 70% | -12 到 5 | -3 到 6 |

自然饱和度:

| 开启概率 | vibrance | saturation |
|---:|---:|---:|
| 50% | -8 到 0 | -10 到 -2 |

目的: 头发可协调变色，但不要抢视觉中心。

### 11.3 Clothing

服装是主色核心。目标:

```text
primaryHue ±8°
```

服装读取 `lightnessBias` 与 `saturationBias` 的编码档位。

#### Grayscale 档

| 参数 | 范围 |
|---|---:|
| hue | 0 |
| saturation | -100 |
| lightness | 深灰 -25..-10 / 中灰 -8..8 / 亮灰 10..25 |
| BrtCtr | 按亮度分支设置 |
| Curves | DeepSafe / HighlightSafe / SoftContrast |
| Vibrance | vibrance -25..-10, saturation -30..-15 |
| ColorBalance / PhotoFilter | 不开 |

#### 非灰度档 HSL

| 亮度档 | lightness |
|---|---:|
| UltraDark | -40 到 -22 |
| Dark | -22 到 -8 |
| Mid | -6 到 8 |
| Bright | 4 到 14 |
| UltraBright | 14 到 24 |

| 饱和档 | saturation |
|---|---:|
| Low | -60 到 -35 |
| Mid | -40 到 -15 |
| High | -5 到 5 |

亮档如果抽到 Low 饱和，会强制提升到 Mid，避免亮低饱变成过曝白片。

#### BrtCtr

| 亮度档 | brightness | contrast |
|---|---:|---:|
| UltraDark | -22 到 -12 | 15 到 28 |
| Dark | -12 到 -3 | 8 到 20 |
| Mid | -4 到 5 | 2 到 14 |
| Bright | 0 到 5 | 0 到 10 |
| UltraBright | 4 到 10 | 2 到 10 |

#### 曲线

| 亮度档 | 曲线 |
|---|---|
| UltraDark | UltraDark |
| Dark | DeepSafe |
| Mid | SoftContrast |
| Bright | HighlightSafe |
| UltraBright | UltraBright |

#### Vibrance

当前智能服装整体降饱，不再强推高饱。

| 饱和档 | vibrance | saturation |
|---|---:|---:|
| Low | -15 到 -3 | -15 到 -5 |
| Mid 暗/中 | -8 到 2 | -8 到 2 |
| Mid 亮档 | -5 到 3 | -3 到 5 |
| High | -3 到 8 | -3 到 8 |

#### ColorBalance / PhotoFilter

- Low 饱和不开颜色平衡。
- Mid / High 饱和按 `ClothingTone` 应用偏色。
- PhotoFilter 概率约 25% 到 40%，浓度 3 到 14，暗档略高。

## 12. 智能随机入口

| UI 按钮 | 函数 | 行为 |
|---|---|---|
| 智能当前层 | `smartRandomizeCurrentLayer()` | 当前方案当前层按现有 palette 和 slot 重算。若无 palette 自动生成。 |
| 智能所有层 | `smartRandomizeAllLayers()` | 当前方案强制生成新 palette，所有可见非肤色层按 slot 重算。 |
| 智能可编辑 | `smartRandomizeAllSchemes(false)` | 所有可编辑方案重生 palette 并重算。跳过本体、锁定、已烘焙。 |
| 智能全部 | `smartRandomizeAllSchemes(true)` | 所有非本体非锁定方案重算。已烘焙会降级为可编辑。 |

保护规则:

1. 本体方案 `isBuiltin` 永远不随机。
2. 锁定方案 `locked` 完全跳过。
3. 未勾选 `includeBaked` 时，已烘焙方案跳过。
4. 勾选 `includeBaked` 时，已烘焙方案清空 `layerLutPath`，`isBaked=false`，变成可编辑方案。
5. 肤色保护层跳过。
6. 隐藏层跳过。

完成后:

1. 设置 dirty。
2. 发射 `effectsChanged()`。
3. 如果方案标签变化，发射 `schemesChanged()`。
4. 如果方案重排，发射 `currentSchemeChanged()`。

## 14. 方案体系

### 14.1 `Scheme`

```cpp
struct Scheme {
    QString name;
    bool isBuiltin;
    bool isBaked;
    bool locked;
    QHash<QString, EffectStack> layerEffects;
    QHash<QString, QString> layerLutPath;
    std::optional<SchemePalette> palette;
};
```

字段含义:

| 字段 | 含义 |
|---|---|
| `name` | UI 显示名。 |
| `isBuiltin` | 本体方案，永不可编辑效果。 |
| `isBaked` | 来自 `add_lut` PNG 的已烘焙方案。 |
| `locked` | 锁定后不参与随机。 |
| `layerEffects` | 可编辑方案: 每层一份 `EffectStack`。 |
| `layerLutPath` | 已烘焙方案: 每层绑定 LUT PNG。 |
| `palette` | 智能随机生成的方案级调色盘。 |

### 14.2 方案 0

方案 0 固定为:

```text
方案 0 - 默认本体
```

特性:

1. `isBuiltin = true`。
2. 不参与随机。
3. 不编辑效果。
4. 用于显示原始外观。

### 14.3 已烘焙方案

已烘焙方案来自 `add_lut` PNG。渲染时走 `recolor.hlsl` 路径，不走 effect_chain 参数。

当执行 `随机全部`、`智能全部`、`智能+随机`、`重置全部` 并包含已烘焙时:

```text
isBaked = false
layerLutPath.clear()
确保每层 layerEffects 存在
```

这会丢失原 add_lut 引用，所以 UI 会弹确认。

### 14.4 锁定方案

`locked = true` 后:

1. 所有随机操作跳过。
2. 方案重排时锁定方案排在本体之后、未锁方案之前。
3. 重排会重命名方案编号前缀。

## 16. GPU 渲染与 LUT 烘焙

### 16.1 实时渲染路径

运行时每层可能走三条路径:

| 条件 | Shader 路径 |
|---|---|
| `EffectStack` 非 identity | `effect_chain.hlsl` |
| 绑定 `add_lut` 且 `useLut` | `recolor.hlsl` |
| 无效果 | `fullscreen_quad.hlsl` 透传 |

### 16.2 effect_chain

`effect_chain.hlsl` 通过 cbuffer 接收 `EffectStack` 参数。

关键字段:

| cbuffer 字段 | 含义 |
|---|---|
| `enableMask.x` | 低 7 位效果开关。 |
| `enableMask.y` | 影子保护阈值。 |
| 后续 float4/int4 | 7 效果参数。 |

处理顺序:

```text
HSL
→ BrtCtr
→ Curves
→ ChMix
→ ColorBal
→ PhotoFilter
→ Vibrance
→ ShadowProtect
```

### 16.3 色彩空间

项目在 sRGB / Gamma 空间处理:

1. RTV 使用 `DXGI_FORMAT_R8G8B8A8_UNORM`。
2. 不用 `_SRGB` 格式。
3. Shader 不做 sRGB / linear 转换。
4. Blend 在 Gamma 空间执行。

目的: 更接近 AE 默认合成表现。

### 16.4 HALD-CLUT

项目使用 16×16×16 HALD-CLUT，平铺为 `256×16` PNG。

编码:

```text
R = (x mod 16) * 17
G = y * 17
B = floor(x / 16) * 17
```

运行时 `recolor.hlsl` 按 RGB 坐标反推 LUT 采样，等价于三线性查找。

### 16.5 LUT 烘焙

`LutBaker` 流程:

```text
默认颜色图 assets/颜色图.png
→ 绑定 effect_chain PSMainBake
→ 256×16 离屏 RenderTarget
→ GPU 渲染
→ Staging 纹理回读
→ enforceShadowLock
→ PNG 导出
```

`enforceShadowLock` 强制 `(0,0)` 像素为黑色，避免纯黑输入被浮点误差污染。

## 18. 当前策略结论

### 18.1 推荐批量出方案方式

推荐顺序:

```text
智能全部
→ 人工挑选/锁定好方案
→ 智能+随机补充中间态
→ 对单层微调 HSL / 曲线 / 色彩平衡
→ 导出 add_lut / PNG / GIF
```

### 18.2 什么时候用旧随机

旧随机适合:

1. 快速制造大差异。
2. 临时寻找意外效果。
3. 单层实验。

不适合:

1. 批量生成稳定好看的 27 套皮肤。
2. 需要统一主题的角色方案。
3. 发光/金属/皮肤明显分层的角色。

### 18.3 什么时候用智能随机

智能随机适合:

1. 批量生成统一主题方案。
2. 角色已有人工分块。
3. 服装/裙摆/头发/装饰/武器需要不同材质策略。
4. 需要暗色、高级、低饱、金属质感。

### 18.4 什么时候用智能+随机

智能+随机适合:

1. 智能结果太稳、想增加变化。
2. 旧随机太乱、想保持主题。
3. 批量扩展候选方案。

## 19. 已知注意点

1. `kSchemeStyles` 27 套预设仍保留，但当前 `generatePalette()` 不使用 `schemeIdx` 固定取它们。
2. 旧 UI tooltip 里“按 idx 取 27 风格槽”与当前源码实际行为不完全一致。
3. `saturationBias` / `lightnessBias` 字段注释仍写偏移范围，但当前实际编码为“档位 ×100 + jitter”。
4. 智能随机没有分析原图真实主色相，`hueAbsToShift()` 是把绝对 hue 近似映射为 AE hue shift。
5. 因为没有像素语义识别，`LayerSlot` 准确性依赖人工分层或默认命名约定。
6. 已烘焙方案降级不可自动恢复原 `add_lut` 引用，执行“全部”类操作前需确认。
7. 中文路径下 `windeployqt` 和 gif.h 写文件有兼容问题，项目已通过 `%TEMP%` 中转处理。

4. 后续可加入每层主色分析，把 `targetHue - sourceDominantHue` 替代当前绝对 hue 近似。
5. 对 `Decor02` 可增加面积判断，小面积才允许更亮更饱，大面积自动降级为 `Decor01`。
