```cmake

```

windeployqt 也会拷 Svg / Network 等 dll, 实际不用, 不影响运行.

## 七、构建 / 运行流程

### 7.1 一键脚本 (推荐)

```cmd
cd x:\Lut变色\HighPro_LUT
build.bat            :: Release
build.bat debug      :: Debug
```

`build.bat` 内部:

1. `setlocal` + `chcp 65001` (UTF-8 终端)
2. 校验 `vcvars64.bat` / `qmake.exe` 存在
3. 把 `CMake_64\bin` / `Ninja` / `Qt\bin` 加进 `PATH`
4. `call vcvars64.bat` 进 MSVC 环境
5. `cmake --preset msvc-release`  (configure)
6. `cmake --build build\release --config Release`  (build)
7. `windeployqt --release ...`  (Qt dll 部署到根目录, 走 `%TEMP%` 中转避免中文路径)

完成输出:

```
=== OK ===
双击运行: x:\Lut变色\HighPro_LUT\HighPro_LUT.exe
```

### 7.2 手动构建 (原理)

打开 **`x64 Native Tools Command Prompt for VS 2026`**:

```cmd
cd /d x:\Lut变色\HighPro_LUT
set "PATH=C:\Qt\Tools\CMake_64\bin;C:\Qt\Tools\Ninja;C:\Qt\6.11.1\msvc2022_64\bin;%PATH%"

cmake --preset msvc-release
cmake --build build\release --config Release

:: Qt dll 部署 (windeployqt 不支持中文路径, 借 TEMP 中转)
mkdir "%TEMP%\hp_deploy"
copy /y HighPro_LUT.exe "%TEMP%\hp_deploy\"
windeployqt --release --no-translations --no-system-d3d-compiler --no-opengl-sw "%TEMP%\hp_deploy\HighPro_LUT.exe"
xcopy /y /e /q "%TEMP%\hp_deploy\*" .\
rmdir /s /q "%TEMP%\hp_deploy"
```

### 7.3 增量构建 (改源码 / shader 后最快)

```cmd
cd /d x:\Lut变色\HighPro_LUT
set "PATH=C:\Qt\Tools\CMake_64\bin;C:\Qt\Tools\Ninja;%PATH%"
cmake --build build\release --config Release
```

Ninja 增量, 几秒. Shader 改了 → AUTORCC 重 generate `qrc_assets.cpp` → 链 EXE.

### 7.4 全清重建

```cmd
cd /d x:\Lut变色\HighPro_LUT
rmdir /s /q build
del HighPro_LUT.exe
build.bat release
```

## 十一、常见环境问题排查

| 现象 | 排查 |
|------|------|
| `[ERROR] vcvars64.bat not found` | VS 2026 没装在默认路径; 改 `build.bat` L16 `VSDEVCMD` |
| `[ERROR] Qt not found at C:\Qt\6.11.1\...` | Qt 装在别的版本目录; 改 `build.bat` L17 `QT_BIN` 与 `CMakeLists.txt` L38 默认前缀 |
| `cmake: 'Ninja' generator not found` | `Ninja\ninja.exe` 不在 PATH | 用 `build.bat` 自动加, 或手动 set PATH |
| `error C2001 / C4828: 字符无效` | 源码非 UTF-8 编码 | 全部存为 UTF-8 (with/without BOM 都行) |
| `LNK2019 unresolved external D3D...` | 没链 `d3d11/dxgi/d3dcompiler` | CMakeLists L137-139 已在, 检查是否被误删 |
| `cannot find Qt6Core.dll 启动失败` | windeployqt 没跑 / dll 不在同目录 | 重跑 `build.bat`, 或手动 `windeployqt EXE` |
| `qt.qpa.plugin: Could not find Qt platform plugin "windows"` | `platforms/qwindows.dll` 缺失 | 同上 |
| 中文路径 windeployqt 报错 | windeployqt 不支持中文路径 | `build.bat` 已处理 (走 `%TEMP%` 中转), 手动操作时模拟 |
| EXE 启动黑屏 | D3D 创建失败 / 显卡驱动太旧 | 看 stderr (改 build.bat 把 EXE 通过 cmd 跑), 升级显卡驱动 |
| 4090 + 集显双卡 NVIDIA 控制面板 让 EXE 走独显 | 性能优 | 项目主目标是 1080p / 60 FPS, 集显也跑得动 |

## 十三、未来变化 (M10 静态打包)

`HIGHPRO_STATIC_RUNTIME=ON` (CMakePreset `msvc-release-static`) 启用后:

- MSVC CRT: `/MT` (静态) — EXE 不依赖 vcruntime140.dll
- Qt: 一期仍动态 (Qt 静态版需自己编源码, 周期 1~2 天)
- 目标: 单 EXE 25~40 MB, 用户机零依赖运行

切换计划: M10 阶段评估 Qt 静态编译收益 vs 维护成本.
