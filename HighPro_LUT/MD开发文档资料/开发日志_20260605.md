# 开发日志 2026-06-05

## 颜色 LayerSlot 与资源树交互优化

### 1. 三分类 LayerSlot 方案落地

本次新增颜色 LayerSlot，并将资源树右键菜单整理为三类：

1. `部件 LayerSlot（旧逻辑）`
2. `单层颜色 LayerSlot`
3. `全层颜色 LayerSlot`

核心原则：

```text
部件 LayerSlot 负责材质/部件语义
颜色 LayerSlot 负责目标色调
旧随机逻辑不受影响
Auto 完全回退旧智能随机
```

新增颜色类型：

```text
红 / 橙 / 黄 / 绿 / 青 / 蓝 / 紫 / 粉 / 黑 / 白 / 银 / 灰 / 自动
```

### 2. 数据结构

新增 `LayerColorSlot`：

```cpp
enum class LayerColorSlot : int {
    Auto = 0,
    Red,
    Orange,
    Yellow,
    Green,
    Cyan,
    Blue,
    Purple,
    Pink,
    Black,
    White,
    Silver,
    Gray,
};
```

`Project` 新增：

```cpp
QHash<QString, LayerColorSlot> layerColorSlots;
```

工程保存新增 JSON 字段：

```json
"layerColorSlots": {
  "num_00": "Blue",
  "num_01": "White"
}
```

兼容逻辑：

- 旧工程无 `layerColorSlots` 时默认 `Auto`
- 未知颜色字符串读取为 `Auto`
- 保存时不写 `Auto`

### 3. 智能随机接入

新增 `randomizeStackBySlot` 重载：

```cpp
void randomizeStackBySlot(EffectStack& s,
                          LayerSlot slot,
                          LayerColorSlot colorSlot,
                          const SchemePalette& palette,
                          quint32 seed = 0);
```

行为：

- `colorSlot == Auto`：完全走旧 `randomizeStackBySlot(slot, palette)`
- 指定颜色：先按旧部件规则生成，再用颜色 Slot patch 目标 hue/saturation/lightness/curve/filter

旧随机 `randomizeStack()` 不读取 `LayerColorSlot`，避免破坏旧功能。

### 4. 单层/全层颜色逻辑

新增控制器接口：

```cpp
void setLayerColorSlot(const QString& layerKey, LayerColorSlot slot);
void setAllLayerColorSlots(LayerColorSlot slot);
```

互斥规则：

```text
一个层级只显示/保留一个预设语义
```

实现：

- 选择部件 LayerSlot：清除该层颜色 Slot
- 选择单层颜色 LayerSlot：清除该层部件 Slot 与肤色保护
- 选择全层颜色 LayerSlot：清空全部部件 Slot 与肤色保护，再给所有层写入同一颜色
- 选择 Auto：清除对应颜色记录，回退旧逻辑

### 5. 资源树交互修正

修正资源树点击行为：

- 点击文字/非勾选区域：只选中该层
- 点击勾选框区域：显示/隐藏该层
- 拖扫显隐只在勾选框区域触发

避免之前点击文字也触发显隐，导致选择逻辑混乱。

### 6. 资源树显示优化

### 7. 右键菜单样式

## 细化方案弹窗 Undo/Redo 体验优化

### 1. 问题一：Ctrl+Z 后通道跳回 RGB

**现象**: 在曲线编辑器的 `R / G / B` 通道修改后,按 `Ctrl+Z`,数据虽然回退,但通道下拉框跳回 `RGB` 主通道,看到的不是"被撤销操作所处通道"的曲线。`Ctrl+Y` 重做也是同样情况。

**根因**: `SchemeRefineDialog` 的 undo 步只快照了 `QHash<QString, EffectStack>` 数据,**不含 UI 上下文**(当前选中层、当前曲线通道)。`localUndo` 调 `rebuildCurrentLayerPanel` → `buildEffectPanel` 是从零重建,combo 默认从 0 开始,因此通道永远跳回 RGB。

**方案**: 把"被撤销操作发生时的层 + 各层曲线通道"也快照进 undo 步,撤销/重做时一并还原 UI。

数据结构变更 (`SchemeRefineDialog.h`):

```cpp
struct LocalUndoStep {
    QHash<QString, EffectStack> effects;          // 全层 EffectStack
    QString                     activeLayerKey;   // 该步发生时选中的层
    QHash<QString, int>         curveChannel;     // 各层"上次查看"的曲线通道 0..3
};
QVector<LocalUndoStep> m_localUndo;
QVector<LocalUndoStep> m_localRedo;

QHash<QString, int> m_layerCurveCh;     // 跨层切换记忆 + panel 重建时恢复 combo
QString             m_sessionStartLayerKey;
QHash<QString, int> m_sessionStartCurveCh;
```

关键改动:

1. `buildEffectPanel` 曲线段:combo 信号改为先写 `m_layerCurveCh[layerKey] = ch` 再 `editor->setChannel(ch)`;editor 创建后用 `m_layerCurveCh.value(layerKey, 0)` 恢复通道。
2. `scheduleApplyToProject` 启动会话时同步记录 `m_sessionStartLayerKey` 与 `m_sessionStartCurveCh`。
3. `commitDragSession` 入栈时构造完整 `LocalUndoStep`。
4. `localUndo` / `localRedo` 还原 effects + `m_layerCurveCh` + 切回 `activeLayerKey` 行,再走 `rebuildCurrentLayerPanel`,combo 自动恢复到操作发生时的通道。

### 2. 问题二:`Ctrl+Z` 需点两次才能回退一步

**现象**: 在曲线上加一个新点并拖动后,`Ctrl+Z` 第一次只回到"加点完成、拖动前"的位置,要按第二次才回到"加点前"。`Ctrl+Y` 同理。

**根因**: `CurveEditor::mousePressEvent` 加点路径 (左键空白处) 在 `emit curvesChanged()` 之后**立即** `emit editingFinished()`,然后又设 `m_dragIdx = 新点`。事件流:

| 阶段 | curvesChanged | editingFinished | 入栈数 |
|---|---|---|---|
| 加点 (press 空白) | 启 session(起点=加点前) | **commit → push 1 步** | 1 |
| 拖动新点 (move) | 重启 session(起点=加点后) | — | — |
| 释放 (release) | — | commit → push 1 步 | 2 |

一次连贯的"加点 + 拖动"动作产生了 **2 个 undo 步**。

**修复**: `CurveEditor.cpp` 加点路径删除立即提交,统一由 `mouseReleaseEvent` 提交一次。

```cpp
// 加点后不再 emit editingFinished —— m_dragIdx 已就位,
// 用户后续可能立即拖动新点; 由 mouseReleaseEvent 统一发一次,
// 保证"加点 + 拖动"是一个 undo 步.
```

回归覆盖:

- 加点不拖直接松开 → release 处 `m_dragIdx >= 0` → 1 步 ✓
- 加点 + 拖动 → 1 步(起点=加点前,终点=拖动后) ✓
- 删点(右键) / `resetCurrent` / `resetAll` 路径不接拖动,保留 `editingFinished` 立即提交 ✓
- 拖动已有点路径不变 ✓
