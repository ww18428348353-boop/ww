# 开发日志 2026-06-05

## 颜色 LayerSlot 与资源树交互优化

### 1. 三分类 LayerSlot 方案落地

本次新增颜色 LayerSlot，并将资源树右键菜单整理为三类：

1. `部件 LayerSlot（旧逻辑）`
2. `单层颜色 LayerSlot`
3. `全层颜色 LayerSlot`

核心原则：

```text
部件 LayerSlot 负责材质/部件语义
颜色 LayerSlot 负责目标色调
旧随机逻辑不受影响
Auto 完全回退旧智能随机
```

新增颜色类型：

```text
红 / 橙 / 黄 / 绿 / 青 / 蓝 / 紫 / 粉 / 黑 / 白 / 银 / 灰 / 自动
```

### 2. 数据结构

新增 `LayerColorSlot`：

```cpp
enum class LayerColorSlot : int {
    Auto = 0,
    Red,
    Orange,
    Yellow,
    Green,
    Cyan,
    Blue,
    Purple,
    Pink,
    Black,
    White,
    Silver,
    Gray,
};
```

`Project` 新增：

```cpp
QHash<QString, LayerColorSlot> layerColorSlots;
```

工程保存新增 JSON 字段：

```json
"layerColorSlots": {
  "num_00": "Blue",
  "num_01": "White"
}
```

兼容逻辑：

- 旧工程无 `layerColorSlots` 时默认 `Auto`
- 未知颜色字符串读取为 `Auto`
- 保存时不写 `Auto`

### 3. 智能随机接入

新增 `randomizeStackBySlot` 重载：

```cpp
void randomizeStackBySlot(EffectStack& s,
                          LayerSlot slot,
                          LayerColorSlot colorSlot,
                          const SchemePalette& palette,
                          quint32 seed = 0);
```

行为：

- `colorSlot == Auto`：完全走旧 `randomizeStackBySlot(slot, palette)`
- 指定颜色：先按旧部件规则生成，再用颜色 Slot patch 目标 hue/saturation/lightness/curve/filter

旧随机 `randomizeStack()` 不读取 `LayerColorSlot`，避免破坏旧功能。

### 4. 单层/全层颜色逻辑

新增控制器接口：

```cpp
void setLayerColorSlot(const QString& layerKey, LayerColorSlot slot);
void setAllLayerColorSlots(LayerColorSlot slot);
```

互斥规则：

```text
一个层级只显示/保留一个预设语义
```

实现：

- 选择部件 LayerSlot：清除该层颜色 Slot
- 选择单层颜色 LayerSlot：清除该层部件 Slot 与肤色保护
- 选择全层颜色 LayerSlot：清空全部部件 Slot 与肤色保护，再给所有层写入同一颜色
- 选择 Auto：清除对应颜色记录，回退旧逻辑

### 5. 资源树交互修正

修正资源树点击行为：

- 点击文字/非勾选区域：只选中该层
- 点击勾选框区域：显示/隐藏该层
- 拖扫显隐只在勾选框区域触发

避免之前点击文字也触发显隐，导致选择逻辑混乱。

### 6. 资源树显示优化

LayerTree 现在只显示一个有效前缀：

1. 肤色保护优先显示盾牌
2. 否则显示颜色 Slot 图标
3. 否则显示部件 Slot 图标
4. Auto 不显示图标

肤色保护层显示：

- 蓝色盾牌 `🛡️`
- 层级文字黄色
- 加粗
- 字体大小与普通层一致

### 7. 右键菜单样式

`标为肤色保护层` / `取消肤色保护层` 改用 `QWidgetAction + QPushButton` 实现，解决 Qt 原生 `QAction` 富文本颜色不稳定问题。

视觉效果：

- 黄色文字
- 加粗
- hover 时保持菜单高亮背景
