# 👕 服装智能随机颜色优化计划

> 项目: `HighPro_LUT`  
> 主题: 👕 Clothing / 服装层智能随机颜色优化  
> 目标: 让服装随机更接近高级暗色配色：暗蓝、暗红、暗紫、暗绿、暗黑灰；暗的更暗但不死黑，亮的更亮但不大面积曝白。  
> 使用场景: 用户每次智能随机生成约 14~21 个方案，可多次点击累计 100~200+ 个候选，并从中筛选 2~3 个优秀方案。

---

## 一、优化目标

服装层的随机目标不是“颜色越随机越好”，而是：

```text
暗的更暗，但不要死黑
亮的更亮，但不要大面积曝白
整体偏暗蓝 / 暗红 / 暗紫 / 暗绿 / 暗黑灰
```

这属于 **暗调高级配色**，不是普通 HSL 随机。

核心方向：

```text
服装主体压暗
保留高光
提高中高光层次
限制白场
限制黑场
使用暗色 hue 池
```

最终视觉应该是：

```text
暗部沉下去
轮廓还看得见
亮边有高光
布料细节不糊
整体颜色不荧光
```

## 二、当前截图问题判断

从当前 EXE 智能随机截图看，随机效果已经比旧版更统一，但服装层仍有几个问题。

### 2.1 有些颜色太亮、太粉、太黄

当前部分方案出现：

```text
亮黄
亮青
亮粉
亮紫
```

这些颜色可以作为“稀有亮色”，但不能占多数。

如果目标是暗蓝、暗红、暗紫、暗绿、暗黑灰，服装主色池不应该全 hue 平均随机，而应该明显偏暗色。

---

### 2.2 暗色方案有些接近死黑

有些暗绿黑、暗紫黑方案有气质，但风险是：

```text
角色缩小后容易糊成一团
服装暗部和背景分离不够
```

暗色要保留：

```text
边缘光
中间调
局部高光
```

不能只靠：

```text
brightness -20
lightness -30
```

直接压暗。

---

### 2.3 亮部有些接近泛白

白蓝、黄白方案中，如果大面积服装整体变亮，会出现：

```text
白布面积过大
纹理细节丢失
发光感变成曝白
```

服装亮不是整体提亮，而是：

```text
暗部保持暗
中间调稍提
高光受限
```

这更适合用曲线控制，而不是只靠 `lightness +`。

## 三、服装随机改成“暗色主题池”

不要让 `Clothing` 主色从 `0~360` 全范围均匀随机。

建议给 👕 服装单独做主色池。

### 3.1 推荐服装主色池

```cpp
enum class ClothingTone {
    DarkBlue,
    DarkRed,
    DarkPurple,
    DarkGreen,
    DarkGray,
    DeepTeal,
    DarkBrown,
    LightNoble,
    RareVivid
};
```

---

### 3.2 推荐权重

| 服装色系 | Hue 范围 | 权重 | 备注 |
|---|---:|---:|---|
| 暗蓝 | 215°~240° | 22% | 最稳，高级 |
| 暗紫 | 265°~290° | 20% | 魔幻感强 |
| 暗红 | 350°~15° | 16% | 黑红、赤红 |
| 暗绿 | 115°~155° | 14% | 墨绿、幽绿 |
| 暗青 | 180°~205° | 10% | 冰冷、幽灵 |
| 暗黑灰 | 任意 / 低饱和 | 10% | 黑白灰高级方案 |
| 暗棕 / 铜黑 | 25°~45° | 5% | 古风、皮革 |
| 亮色稀有 | 全 hue 控制 | 3% | 彩蛋，不要多 |

这样随机 100~200 个时，大概率能刷出目标暗色高级方案。

---

## 四、服装目标颜色不能只用 Hue

对于服装，建议目标色使用：

```text
Hue + Saturation + Value/Lightness
```

而不是只改色相。

目标：

```text
Hue = 从暗色池抽
Saturation = 中低饱和
Value / Lightness = 中低亮度
```

### 4.1 暗色服装推荐目标范围

| 色系 | Saturation | Value / Lightness | 备注 |
|---|---:|---:|---|
| 暗蓝 | 35~65 | 22~42 | 避免纯黑蓝 |
| 暗紫 | 30~60 | 20~40 | 避免荧光紫 |
| 暗红 | 40~70 | 22~42 | 保留红色识别 |
| 暗绿 | 30~55 | 20~38 | 避免脏绿 |
| 暗青 | 35~60 | 24~44 | 可以稍亮 |
| 暗黑灰 | 5~20 | 18~35 | 偏冷灰最好 |
| 暗棕 | 25~50 | 22~38 | 避免土黄 |
| 亮色稀有 | 35~65 | 45~65 | 不能大面积曝白 |

重点：

```text
暗色不是 brightness 一路往下压
而是目标亮度维持在 20~42 左右
```

## 五、暗的更暗，但不要死黑

不要简单使用：

```text
brightness = -30
lightness = -30
```

这会把服装压死。

应该使用：

```text
暗部压低 + 黑位保护
```

### 5.1 推荐曲线：DeepSafe Curve

用于暗色服装。

```text
输入 0   → 输出 8
输入 32  → 输出 22
输入 96  → 输出 78
输入 160 → 输出 152
输入 220 → 输出 225
输入 255 → 输出 248
```

解释：

```text
0 不输出 0，而是输出 8 → 不死黑
32 输出 22 → 暗部更暗
96 输出 78 → 中暗部下压
160 输出 152 → 中高光基本保留
220 输出 225 → 亮部略亮
255 输出 248 → 防止爆白
```

曲线点：

```cpp
stack.curves.master = {
    {0, 8},
    {32, 22},
    {96, 78},
    {160, 152},
    {220, 225},
    {255, 248}
};
stack.enabled[EffectStack::ECurves] = true;
```

这条曲线非常关键。它比单纯亮度 / 对比度更适合暗调服装。

## 六、亮的更亮，但不要大面积曝白

亮部不要靠：

```text
lightness +20
brightness +20
```

而要用：

```text
高光压顶 + 中高光提升
```

### 6.1 推荐曲线：HighlightSafe Curve

适合白蓝、白金、浅粉方案。

```text
输入 0   → 输出 10
输入 48  → 输出 45
输入 128 → 输出 138
输入 200 → 输出 218
输入 240 → 输出 238
输入 255 → 输出 248
```

解释：

```text
中高光变亮
最高亮不超过 248
避免大面积白爆
暗部仍有层次
```

曲线点：

```cpp
stack.curves.master = {
    {0, 10},
    {48, 45},
    {128, 138},
    {200, 218},
    {240, 238},
    {255, 248}
};
stack.enabled[EffectStack::ECurves] = true;
```

## 七、👕 Clothing 的 HSL 参数建议

当前可能类似：

```text
primaryHue ±8
sat -18..+8
light -6..+6
```

对“暗色高级服装”不够强。

建议服装分 4 个 tone profile。

---

### 7.1 DarkRich 暗色高级

用于：

```text
暗蓝
暗紫
暗红
暗绿
暗青
```

参数：

```text
Hue: targetHue ± 5°~8°
Saturation: -12 到 +10
Lightness: -18 到 -6
Brightness: -8 到 -2
Contrast: +8 到 +18
Curve: DeepSafe
Vibrance: -8 到 +5
```

特点：

```text
暗下去
不死黑
高光还在
```

---

### 7.2 DarkNeutral 暗黑灰

用于：

```text
暗黑灰
黑银
黑金
黑红
```

参数：

```text
Hue: 可不动或偏冷 220°±10°
Saturation: -35 到 -18
Lightness: -20 到 -8
Brightness: -10 到 -3
Contrast: +10 到 +22
Curve: DeepSafe
Vibrance: -20 到 -8
```

特点：

```text
低饱和
有黑色气质
但不是纯黑
```

---

### 7.3 LightNoble 高级亮色

用于：

```text
白蓝
白金
珍珠白
浅粉
```

参数：

```text
Hue: targetHue ±6°
Saturation: -20 到 -5
Lightness: +4 到 +12
Brightness: +2 到 +8
Contrast: -4 到 +8
Curve: HighlightSafe
Vibrance: -12 到 0
```

特点：

```text
亮
但不曝白
更像白金 / 冰蓝 / 珍珠，不是纯白块
```

---

### 7.4 RareVivid 稀有彩色

只给少数方案。

参数：

```text
Hue: targetHue ±5°
Saturation: 0 到 +14
Lightness: -4 到 +6
Brightness: -2 到 +4
Contrast: +4 到 +12
Curve: SoftContrast
Vibrance: +0 到 +8
```

限制：

```text
占比最多 3%~5%
不能大面积出现在服装主色
```

## 九、SchemePalette 应加入 clothingTone

当前 `SchemePalette` 可能是：

```cpp
struct SchemePalette {
    float primaryHue;
    float secondaryHue;
    float accentHue;
    float accent2Hue;
    float glowHue;
    MetalTone metalTone;
    StyleMood mood;
    float bias;
};
```

建议加入：

```cpp
enum class ClothingTone {
    DarkBlue,
    DarkRed,
    DarkPurple,
    DarkGreen,
    DarkGray,
    DeepTeal,
    DarkBrown,
    LightNoble,
    RareVivid
};

struct SchemePalette {
    float primaryHue;
    float secondaryHue;
    float accentHue;
    float accent2Hue;
    float glowHue;

    MetalTone metalTone;
    StyleMood mood;
    ClothingTone clothingTone;

    float clothingSatTarget;
    float clothingValueTarget;
    float bias;
};
```

这样 `applyClothing()` 可以知道当前服装应该走什么策略。

## 十、14~21 个方案怎么分布

用户每次智能随机大概生成 14~21 个方案。建议每次随机保证大多数是暗色，少数亮色。

### 10.1 一次 21 个方案推荐分布

| 数量 | 类型 |
|---:|---|
| 4 | 暗蓝 |
| 4 | 暗紫 |
| 3 | 暗红 |
| 3 | 暗绿 |
| 3 | 暗黑灰 |
| 2 | 暗青 / 深海 |

| 1 | 稀有彩色 |

这样点一次，大约 19 个符合暗色方向。

## 十一、多次随机 100~200 个时的策略

用户可以多点很多次，然后筛 2~3 个。算法应该：

```text
保证安全边界
提高好看概率
保留一定随机性
```

建议引入 reroll variance：

```cpp
palette.primaryHue += random(-6, +6);
palette.clothingSatTarget += random(-8, +8);
palette.clothingValueTarget += random(-5, +5);
palette.accentHue += random(-15, +15);
```

但必须 clamp：

```text
暗色 Value 永远不低于 18
高光输出永远不超过 248
服装 Saturation 不超过 70
黑灰 Saturation 不超过 22
```

## 十二、服装颜色安全边界

P0 即使不做 `LayerStats`，也可以通过参数边界保证基础质量。

### 12.1 防死黑

```text
HSL lightness 最低 -22
Brightness 最低 -12
Curve 黑位输出最低 8
shadowProtectThreshold 保持 8~12
```

不要：

```text
lightness -30
brightness -25
curve 0→0
```

---

### 12.2 防曝白

```text
HSL lightness 最高 +12
Brightness 最高 +8
Curve 255→248
PhotoFilter density 服装最多 12~18
Vibrance saturation 服装最多 +8
```

不要：

```text
lightness +25
brightness +20
photoFilter density 40+
```

---

### 12.3 防荧光

```text
服装 Saturation 最多 +12
Vivid 只占 3%~5%
暗绿 sat 上限更低
黄色 / 青色服装减少概率
```

---

### 12.4 防发灰

```text
暗色不要 saturation 全部 -30
暗蓝 / 暗紫 / 暗红保留 35~60 目标饱和
暗黑灰才低饱和
```

## 十三、`applyClothing()` 推荐实现逻辑

伪代码：

```cpp
static void applyClothing(EffectStack& s,
                          const SchemePalette& p,
                          QRandomGenerator& rng)
{
    preserveShadowAndReset(s);

    switch (p.clothingTone) {
    case ClothingTone::DarkBlue:
    case ClothingTone::DarkRed:
    case ClothingTone::DarkPurple:
    case ClothingTone::DarkGreen:
    case ClothingTone::DeepTeal:
        applyDarkRichClothing(s, p, rng);
        break;

    case ClothingTone::DarkGray:
        applyDarkNeutralClothing(s, p, rng);
        break;

    case ClothingTone::LightNoble:
        applyLightNobleClothing(s, p, rng);
        break;

    case ClothingTone::RareVivid:
        applyRareVividClothing(s, p, rng);
        break;
    }
}
```

### 13.1 DarkRichClothing 示例

```cpp
static void applyDarkRichClothing(EffectStack& s,
                                  const SchemePalette& p,
                                  QRandomGenerator& rng)
{
    s.enabled[EffectStack::EHsl] = true;
    s.hsl.hue = jitterHueToSigned(p.primaryHue, rng.bounded(-6, 7));
    s.hsl.saturation = randRange(rng, -12, 10);
    s.hsl.lightness  = randRange(rng, -18, -6);

    s.enabled[EffectStack::EBrtCtr] = true;
    s.brtCtr.brightness = randRange(rng, -8, -2);
    s.brtCtr.contrast   = randRange(rng, 8, 18);

    s.enabled[EffectStack::ECurves] = true;
    s.curves.master = {
        {0, 8},
        {32, 22},
        {96, 78},
        {160, 152},
        {220, 225},
        {255, 248}
    };

    s.enabled[EffectStack::EVibrance] = true;
    s.vibrance.vibrance = randRange(rng, -6, 6);
    s.vibrance.saturation = randRange(rng, -8, 5);
}
```

注意：当前效果链的 HSL `hue` 是“色相偏移”，不是绝对目标色。P0 没有源层主色分析时，不能精确做到：

```text
hueShift = targetHue - sourceDominantHue
```

P0 可先用 `primaryHue` 作为色相偏移中心，并限制偏移范围。P1 有 `LayerStats` 后，再改成真正目标色校正。

## 十四、P0 没有 LayerStats 时如何更像目标色

当前没有源图主色分析，所以不能真正“把服装变成暗蓝”。只能通过：

```text
HSL hue shift
ColorBalance
PhotoFilter
```

共同逼近目标色。

### 14.1 暗蓝服装

策略：

```text
HSL hue 偏向蓝区
ColorBalance 阴影加蓝、加少量绿
PhotoFilter 冷色低密度
```

参数：

```text
colorBal.sB +8~+16
colorBal.mB +4~+10
colorBal.hB +2~+6
photoFilter RGB = 冷蓝
density = 6~12
```

---

### 14.2 暗红服装

策略：

```text
阴影加红少量蓝
中间调加红
高光加暖
```

参数：

```text
sR +8~+16
sB +2~+8
mR +6~+14
hR +3~+8
photoFilter 暖红低密度 5~10
```

---

### 14.3 暗紫服装

策略：

```text
阴影加蓝红
中间调加蓝红
```

参数：

```text
sR +6~+12
sB +8~+16
mR +4~+10
mB +6~+12
photoFilter 紫色低密度 6~12
```

---

### 14.4 暗绿服装

策略：

```text
阴影加绿少量蓝
中间调加绿
```

参数：

```text
sG +8~+14
sB +2~+8
mG +5~+12
photoFilter 绿 / 青低密度 5~10
```

---

### 14.5 暗黑灰服装

策略：

```text
降低饱和
阴影偏冷
高光保银白
```

参数：

```text
sB +4~+10
mB +2~+6
hR/G/B 接近均衡
photoFilter density 0~6
```

## 十五、服装用 PhotoFilter 要克制

服装面积大，照片滤镜不能太重。

推荐：

```text
Clothing PhotoFilter density 0~12
LightNoble 可 8~16
DarkRich 可 4~10
DarkGray 可 0~6
```

不要：

```text
density 25+
```

否则服装会整体染色发脏。

## 十八、人工筛选标准

用户在 100~200 个候选里挑 2~3 个时，可按下面标准判断。

### 18.1 好方案特征

```text
缩小看，角色轮廓仍清楚
服装暗部不糊成背景
胸口 / 裙摆 / 袖口有亮边
主色明显属于暗蓝 / 暗红 / 暗紫 / 暗绿 / 暗灰之一
装饰色没有抢服装主体
皮肤和头发不被污染
```

---

### 18.2 坏方案特征

```text
服装一团死黑
裙摆大面积白爆
颜色像荧光笔
黄绿过亮
粉紫大面积过艳
服装和装饰全都同样鲜艳
```

## 十九、P0 实际改动优先级

## 二十、一句话方案

👕 服装随机优化重点：

```text
不要全 hue 随机
改用暗色主色池
用 DeepSafe 曲线压暗但保黑位
用 HighlightSafe 曲线提亮但压白场
服装 PhotoFilter 克制
Vivid 只做极少数彩蛋
```

用户多点几次智能随机时，系统会在：

```text
暗蓝 / 暗红 / 暗紫 / 暗绿 / 暗黑灰/等等等颜色
```

这些方向里大量生成安全候选。然后用户从 100~200 个里挑 2~3 个，命中率会明显高于当前逻辑。
