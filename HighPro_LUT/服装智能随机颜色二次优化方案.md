# 👕 服装智能随机颜色二次优化方案

> 项目: `HighPro_LUT`  
> 主题: 根据截图二继续优化 👕 服装随机颜色，使其更贴近截图一参考方向。  
> 目标: 减少浅粉、亮紫、亮蓝、亮黄等糖果色倾向；强化暗蓝黑、暗紫黑、黑红、墨绿黑、黑银、冰蓝白、白绿、黑白灰等高级配色。

---

## 一、核心判断

截图二的服装随机还没有完全贴近截图一。

当前主要问题：

```text
服装主色太浅
粉 / 紫 / 蓝偏糖果色
暗色不够沉
亮色面积过大
黑位不够有质感
```

截图一参考方向是：

```text
暗主体 + 局部亮边 + 低饱和高级色
```

截图二当前更像：

```text
浅色整体染色
```

所以 👕 服装层需要从：

```text
彩色随机
```

改成：

```text
暗色主体随机 + 小面积亮边
```

## 二、截图一的服装颜色逻辑

截图一上排参考方案规律很好。

| 参考方案 | 服装主体 | 高光 | 整体感觉 |
|---|---|---|---|
| 原色 | 米白 / 黑 / 金 | 暖白 | 本体质感 |
| 暗绿黑 | 近黑绿 | 暗绿边光 | 高级暗色 |
| 暗紫黑 | 深紫 / 黑 | 紫灰高光 | 魔幻暗色 |
| 冰蓝白 | 冷白 / 冰蓝 | 蓝白高光 | 亮但不曝 |
| 红黑 | 深红 / 黑 | 红橙亮边 | 战斗感 |
| 白绿 | 象牙白 / 淡绿 | 绿白高光 | 清冷自然 |
| 黑白灰 | 黑 / 白 / 灰 | 银白 | 高级灰阶 |

共同点：

```text
大面积不是高饱和颜色
暗色有黑位
亮色有灰度
高光面积小
服装主色和阴影绑定
```

## 三、截图二当前问题

截图二里部分方案出现：

```text
浅紫偏亮
粉色偏亮
蓝色偏亮
浅绿偏亮
服装主体没有足够暗部
```

问题不是颜色不够多，而是：

```text
颜色整体太浮
亮度过高
饱和度偏糖果
暗部没有压住
```

需要调整方向：

```text
减少浅粉 / 亮紫 / 亮蓝比例
增强暗蓝 / 暗紫 / 暗红 / 暗绿 / 暗灰比例
压暗服装核心
保留裙摆和边缘高光
```

## 四、服装主色池进一步收窄

之前计划是暗色 80%+。根据截图二效果，建议进一步提高暗色比例：

```text
暗色 90%
亮色 8%
稀有彩色 2%
```

### 4.1 推荐权重

| 类型 | 权重 | 目标 |
|---|---:|---|
| 暗蓝 | 22% | 深海 / 夜蓝 / 蓝黑 |
| 暗紫 | 20% | 紫黑 / 魔纹 / 暗法 |
| 暗红 | 16% | 黑红 / 熔岩 / 魔化 |
| 暗绿 | 14% | 墨绿 / 幽绿 / 毒雾 |
| 暗黑灰 | 12% | 黑银 / 黑金 / 冷灰 |
| 暗青 | 6% | 青黑 / 幽光 |
| 暗棕 | 3% | 铜黑 / 皮革 |
| 高级亮色 | 5% | 冰蓝白 / 白金 / 淡绿白 |
| 稀有彩色 | 2% | 少量粉紫 / 宝石蓝 |

这样可以避免一批方案里大量出现粉、亮紫、亮蓝。

## 六、禁止大面积糖果粉 / 亮紫 / 亮蓝

截图二里粉紫偏多。建议对 `Clothing` 增加颜色禁区。

### 6.1 服装禁区

```text
Hue 285°~330° 且 Saturation 高 → 禁止大面积
Hue 190°~210° 且 Lightness 高 → 降饱和
Hue 50°~75° 黄绿色 → 降亮度
```

### 6.2 规则示例

```cpp
if (slot == LayerSlot::Clothing) {
    if (tone == RareVivid) {
        probability <= 0.02f;
    }

    if (hue >= 285 && hue <= 330) {
        saturationMax = 45;
        valueMax = 42;
    }

    if (hue >= 190 && hue <= 210) {
        saturationMax = 50;
        valueMax = 44;
    }
}
```

粉紫可以留给：

```text
Decor02
Glow
小装饰
```

不要给大面积服装。

## 七、必须加“黑位不死黑”曲线

截图一中暗绿黑、暗紫黑好看，是因为：

```text
主体很暗
但边缘和纹理还在
```

建议所有暗色 `Clothing` 使用这条曲线：

```cpp
s.enabled[EffectStack::ECurves] = true;
s.curves.master = {
    {0, 10},
    {32, 22},
    {80, 62},
    {128, 112},
    {190, 196},
    {230, 232},
    {255, 246}
};
```

效果：

```text
黑位最低 10，不死黑
暗部压下去
中间调更沉
亮部不爆
255 压到 246
```

## 八、暗色方案要加 ColorBalance

P0 没有 `LayerStats`，只靠 HSL 不稳定。

需要 `ColorBalance` 帮服装做暗调染色。

### 8.1 暗蓝

```text
sB +12~+20
mB +6~+14
hB +2~+6
sR -4~0
```

### 8.2 暗紫

```text
sR +6~+12
sB +12~+20
mR +4~+10
mB +8~+14
```

### 8.3 暗红

```text
sR +12~+20
sB +2~+8
mR +8~+16
hR +3~+8
```

### 8.4 暗绿

```text
sG +10~+18
sB +3~+8
mG +6~+14
hG +2~+6
```

### 8.5 暗灰

```text
sB +4~+10

```

## 九、PhotoFilter 继续压低

截图二有些方案像被整体滤镜染浅。服装面积大，滤镜密度要更低。

### 9.1 建议范围

```text
DarkRich: density 3~8
DarkNeutral: density 0~5
LightNoble: density 5~10
RareVivid: density 0~6
```

硬上限：

```text
Clothing density <= 12
```

如果当前是：

```text
10~18
```

建议继续降。

## 十、服装和裙摆要分层处理

截图二里裙摆 / 服装整体一起亮，导致大面积发浅。

建议：

```text
Clothing 更暗
Skirt 稍亮一点
```

这样更接近截图一：

```text
身体核心暗
裙摆边缘 / 下摆有亮度
```

### 10.1 Clothing

```text
Lightness -20~-8
Contrast +12~22
```

### 10.2 Skirt

```text
Lightness -12~+2
Contrast +6~16
Saturation 比 Clothing 低 5~10
```

最终关系：

```text
服装压暗
裙摆保层次
```

## 十四、最终目标效果

下一版截图应该更像：

```text
暗蓝黑
暗紫黑
黑红
墨绿黑
黑银
冰蓝白
白绿
黑白灰
```

而不是：

```text
浅粉
亮紫
亮蓝
亮黄
高饱和糖果色
```

一句话：

```text
👕 服装要从“随机彩色布”改成“暗色高级主体布料”。
```

重点不是增加颜色，而是：

```text
压暗主体
保住高光
降低饱和
减少粉紫亮色概率
让裙摆承担一点亮度
```
