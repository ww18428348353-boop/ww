| 时刻 | 产物 | 内容 |

| 早 | 高性能序列帧角色换色工具EXE开发技术方案研究.md | 用户调研, 比对 Dear ImGui+OpenGL vs Qt+D3D11 |

## 四、关键技术决策时间点

| 日期 | 决策 | 影响 |
|------|------|------|
| 05-30 | 选 Qt6 + D3D11 (放弃 ImGui+OpenGL) | 整个 UI / 渲染基础 |
| 05-30 | HALD-CLUT 16³ 颜色管线 | 4096 色统一编解码, 与 AE/PS 互操作 |
| 05-30 | 影子保护 = 锁 (0,0) 像素 | 角色阴影天然免疫变色 |
| 06-02 | **alpha 合成 Linear → Gamma 切回** | 光晕外溢修复, 视觉与 AE 对齐 |
| 06-02 | TGA Straight Alpha + 不预乘 | 与磁盘原始语义一致 |
| 06-02 | RTV 用 `B8G8R8A8_UNORM` (非 _SRGB) | 字节透传, 不二次 encode |
| 06-02 | GIF 走 Level 2 全局调色板 + Floyd-Steinberg | 256 色 dither, 帧间一致 |
| 06-03 | 随机系统从参数随机 → MaterialRole 半语义分块 | 方案有主次, 不再霓虹色集合 |
| 06-03 | 引入 SchemePalette 方案级调色盘 | 一套方案内多层颜色协调 |
| 06-04 | 静态 Qt + /MT 单 EXE | 24.1 MB, 无 dll, 干净机直跑 |

## 五、踩坑回顾表

| 坑 | 现象 | 根因 | 修复时间 |
|----|------|------|---------|
| 光晕外溢 | EXE 半透明区比 AE/游戏亮 9~23/255 | Linear 空间合成 (凸函数压低中等亮度) | 06-02 |
| sRGB 误转换 | PS 内 `srgbToLinear` + RTV 又 encode | 双重 gamma 转换 | 06-02 |
| 中文路径 stb fopen 失败 | TGA 读不到 | `fopen` 不支持 UTF-8 | M2 期统一改 `QFile::readAll` + `stbi_load_from_memory` |
| GIF 颜色断层 | 渐变发糊 / 帧间闪烁 | 逐帧独立量化 + 无 dither | 06-02 Level 2 全局调色板 |
| 随机方案霓虹色多 | 27 套方案缺设计感 | 每层独立 HSL 随机, 无配色主题 | 06-03 SchemePalette + MaterialRole |
| 服装太亮太粉 | 出现亮黄/亮青/亮粉 | 没有暗调约束 | 06-03 服装专项二次优化 |
| 静态 Qt 编译失败 | vcvars64 报错 | VS 2026 v18 路径变动 + vswhere 不在 PATH | 06-04 脚本中前置加 PATH |
| Qt 源码 configure 异常 | 诡异编码错误 | 源码放在中文目录 | 06-04 改到 `Y:\qt-everywhere-src-6.11.1` |
